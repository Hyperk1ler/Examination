# 状态空间表示法 —— 从零到解题

---

## 一、核心思想

状态空间表示法的本质：**把问题的求解过程，转化为在"状态图"中寻找从起点到终点的路径。**

每一个状态代表问题在某一时刻的快照，操作符让状态发生变化，搜索就是在这些状态之间找到一条通往目标的路。

---

## 二、四元组

$$
\text{问题} = (S, O, S_0, G)
$$

| 符号 | 含义 | 例：传教士问题 |
|:----:|:----:|:--------------:|
| **S** | 所有可能状态的集合 | $(m, c, b)$ 所有合法组合 |
| **O** | 操作符集合（让状态变化的动作） | 船载 i 个传教士+j 个野人过河 |
| **S₀** | 初始状态（问题一开始的样子） | $(3, 3, 1)$ |
| **G** | 目标状态（解的样子） | $(0, 0, 0)$ |

---

## 三、答题三步法

```
① 定义状态表示 —— 决定用什么参数描述一个状态
② 确定 S₀ 和 G  —— 初始是什么？目标是什么？
③ 列出操作符   —— 什么动作能改变状态？受什么约束？
```

### 第①步：定义状态表示

选择能**完整描述问题当前情况**的最小参数集合。

```
常见模式：
S = (参数1, 参数2, 参数3, ...)

传教士问题：S = (左岸传教士数, 左岸野人数, 船的位置)
八数码问题：S = 3×3 矩阵（9个数字的排列）
农夫过河：  S = (农夫位置, 狼位置, 羊位置, 白菜位置)
```

**原则**：参数之间相互独立，参数组合能唯一确定一个状态。

### 第②步：写出初始和目标

直接根据题目描述写出 $S_0$ 和 $G$。

### 第③步：找出所有操作符

操作符 = 动作描述。通常用符号表示：

```
P(i,j)  从左岸到右岸，船上载 i 个传教士、j 个野人
Q(i,j)  从右岸到左岸，船上载 i 个传教士、j 个野人
```

然后加上**约束条件**（安全约束、容量约束等），过滤出合法的操作。

---

## 四、完整例题：从读题到写出答案

### 题目

> 3 个传教士和 3 个野人准备从左岸渡河到右岸。船每次至多可乘 2 人，所有人都会划船。安全约束：在任何一边岸上，野人数不能超过传教士数目（除非该岸传教士数为 0）。请用状态空间表示法描述此问题。

---

### 第①步：定义状态表示

要描述"当前情况"，需要知道三个信息：
1. 左岸还有几个传教士？
2. 左岸还有几个野人？
3. 船在哪边？

所以：

$$
S = (m, c, b)
$$

| 参数 | 含义 | 取值范围 |
|:----:|:----:|:--------:|
| $m$ | 左岸传教士数 | 0, 1, 2, 3 |
| $c$ | 左岸野人数 | 0, 1, 2, 3 |
| $b$ | 船的位置（1=左岸，0=右岸） | 0, 1 |

### 第②步：确定初始和目标

$$
S_0 = (3, 3, 1) \quad\text{（所有人都在左岸，船在左岸）}
$$
$$
G = (0, 0, 0) \quad\text{（所有人都到了右岸，船在右岸）}
$$

### 第③步：定义操作符及约束

**操作符**：$P_{ij}$ 表示船从左岸→右岸，载 $i$ 个传教士、$j$ 个野人。$Q_{ij}$ 表示右岸→左岸。

**约束一：船容量**
$$
1 \leq i + j \leq 2
$$
（不能空船走，也不能超载）

**约束二：船上安全**
$$
i = 0 \quad\text{或}\quad i \geq j
$$
（如果船上有传教士，传教士数必须 ≥ 野人数）

**约束三：两岸安全**（操作执行后要检查）

左岸条件：
$$
m' = 0 \quad\text{或}\quad m' \geq c'
$$

右岸条件（右岸传教士数 $= 3 - m'$，野人数 $= 3 - c'$）：
$$
3 - m' = 0 \quad\text{或}\quad 3 - m' \geq 3 - c'
$$

**列出所有合法的操作**：

按约束一和约束二，可能的载人组合：
$$
(i, j) = (0,1), (0,2), (1,0), (1,1), (2,0)
$$

每种组合对应两个方向的操作：
```
从左岸→右岸：P₀₁, P₀₂, P₁₀, P₁₁, P₂₀
从右岸→左岸：Q₀₁, Q₀₂, Q₁₀, Q₁₁, Q₂₀
```

共 **10 种**合法操作。

---

## 五、操作的效果 —— 状态怎么变

以 $P_{ij}$ 为例（从左岸到右岸）：

$$
(m', c', b') = (m - i,\; c - j,\; 0)
$$

人从左边离开，左岸人数减少；船去了对岸，b = 0。

以 $Q_{ij}$ 为例（从右岸回左岸）：

$$
(m', c', b') = (m + i,\; c + j,\; 1)
$$

人从右边回来，左岸人数增加；船回到左岸，b = 1。

---

## 六、验证操作合法性 —— 两岸安全检查

以初始状态 $(3,3,1)$ 为例，对所有可能的 $P_{ij}$ 进行检查：

```
状态 (3,3,1)，船在左岸

◇ P₀₁（载 0 传教士、1 野人从左到右）：
    新状态 = (3-0, 3-1, 0) = (3, 2, 0)
    左岸：m'=3, c'=2 → 3 ≥ 2 ✅
    右岸：(3-3)=0 → 无传教士 ✅

◇ P₀₂（载 0 传教士、2 野人从左到右）：
    新状态 = (3-0, 3-2, 0) = (3, 1, 0)
    左岸：3 ≥ 1 ✅
    右岸：3-3=0 ✅

◇ P₁₀（载 1 传教士、0 野人从左到右）：
    新状态 = (3-1, 3-0, 0) = (2, 3, 0)
    左岸：2 < 3 ❌ 不安全！
    （左岸野人比传教士多，吃人了）

◇ P₁₁（载 1 传教士、1 野人从左到右）：
    新状态 = (3-1, 3-1, 0) = (2, 2, 0)
    左岸：2 ≥ 2 ✅
    右岸：(3-2)=1, (3-2)=1 → 1 ≥ 1 ✅

◇ P₂₀（载 2 传教士、0 野人从左到右）：
    新状态 = (3-2, 3-0, 0) = (1, 3, 0)
    左岸：1 < 3 ❌ 不安全！
```

结论：从初始状态 $(3,3,1)$ 出发，合法的操作是 **$P_{01}$、$P_{02}$、$P_{11}$**。

## 七、搜索过程示意（扩展）

从 $S_0(3,3,1)$ 出发，第一次扩展后：

```
                 (3,3,1)                 ← S₀
                /   |   \
               /    |    \
            P₀₁   P₀₂   P₁₁
             /      |      \
       (3,2,0)  (3,1,0)  (2,2,0)
```

每个新状态再继续用 $Q$ 操作（从右岸回去的人），如此交替直到抵达 $(0,0,0)$。

---

## 八、考试完整答案格式

```
(1) 状态表示
    S = (m, c, b)
    m: 左岸传教士数 (0≤m≤3)
    c: 左岸野人数 (0≤c≤3)
    b: 船的位置 (1=左岸, 0=右岸)

(2) 初始状态和目标状态
    S₀ = (3, 3, 1)     // 所有人左岸，船左岸
    G  = (0, 0, 0)     // 所有人右岸，船右岸

(3) 操作符
    Pij: 从左岸→右岸，载 i 个传教士、j 个野人
    Qij: 从右岸→左岸，载 i 个传教士、j 个野人

    约束条件：
    ① 1 ≤ i+j ≤ 2
    ② i=0 或 i≥j
    ③ 操作后两岸均需满足：传教士=0 或 传教士≥野人

    合法操作共 10 种：
    左岸→右岸：P₀₁, P₀₂, P₁₀, P₁₁, P₂₀
    右岸→左岸：Q₀₁, Q₀₂, Q₁₀, Q₁₁, Q₂₀
```

## 九、练习题

### 题目：农夫过河问题

> 农夫要将一只狼、一只羊、一筐白菜从左岸运到右岸。船每次只能载农夫和一件物品。约束：农夫不在场时，狼会吃羊，羊会吃白菜。请用状态空间表示法描述此问题。

**提示**：

- 状态最少用几个参数？试着用 4 个二进制参数表示
- 农夫的位置和每件物品的位置
- 操作符是什么？（每次渡河农夫可以带什么？）
- 安全条件是什么？（农夫不在时，哪些组合不能出现？）
