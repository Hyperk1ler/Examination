## 第一部分：简答题核心考点速查（覆盖三年全部考点）

### 一、通信网络基础（第1章）

**1. 现代通信网络五大特征**（2024简答1）
- 多样化：网络类型/结构/业务多样
- 宽带化：5G峰值几十Gb/s，端到端延时几ms，100万用户/km²
- 移动化：移动用户超固话用户
- 泛在化：无处不在，RFID+传感器，自律网
- 可信化：安全性、可生存性、可管理性

**2. 五种拓扑结构**：总线型(易扩充/故障难定位)、星状(灵活/依赖中心)、环状(适合光纤)、树状(分层容错)、网状(最可靠/效率低)

**3. 四种交换技术对比**（核心表）：

| 特征 | 电路交换 | 分组交换 | ATM | IP交换 |
|------|----------|----------|-----|--------|
| 连接 | 面向连接 | 无连接/虚电路 | 面向连接 | 无连接 |
| 粒度 | 时隙 | 变长分组 | 53B信元 | 变长分组 |
| 时延 | 固定小 | 可变 | 低可预测 | 尽力而为 |
| QoS | 最好 | 无 | 好 | 需外部 |

**4. 四种复用技术**：TDM(时分)、FDM(频分)、WDM(波分，DWDM间隔0.8nm)、CDM(码分，需正交S·T=0)

**5. PDH→SDH→OTN演进**：PDH缺点(缺准管难：缺标准光接口、准同步滑码、管理开销少、上下电路难)；SDH速率：STM-1=155M→STM-4=622M→STM-16=2.5G→STM-64=10G

**6. 数据传输链路两类**（2025简答1）：物理传输链路(传比特但不可靠)、数据传输链路(有检测/确认/重传，可靠)

**7. 支撑网三大组成**（2025简答2）：信令网(No.7信令，神经系统)、同步网(时钟同步)、管理网(TMN/SNMP)

**8. 传送网定义**（2023简答2）：为交换节点间提供透明传输通道；含传输介质+交换设备+技术体制(PDH/SDH/OTN)

**9. 通信网络可信化**（2023简答4）：行为可预期可管理；三大属性：安全性、可生存性、可管理性

### 二、No.7信令系统（三年轮流考各级功能）

**四级结构**（必画）：
- 第1级 MTP1：信令数据链路(64Kbit/s，物理电气特性)
- 第2级 MTP2：信令链路功能(分界定位/CRC检错/重传纠错/故障恢复/流量控制) ← 2023简答1
- 第3级 MTP3：信令网功能 = 消息处理(鉴别/分配/路由) + 信令网管理(业务量/链路/路由管理) ← 2024简答3
- 第4级 UP：用户部分(TUP/ISUP/SCCP/TCAP)

**信令单元三种**：MSU(传业务)、LSSU(传链路状态)、FISU(空闲填充)

**局间ISUP信令流程**(2025论述1)：IAM(初始地址)→ACM(地址全，振铃)→ANM(应答，摘机/计费)→通话→REL(释放)→RLC(释放完成)

**信令网组成**：SP(信令点)、STP(信令转接点)、信令链路。结构：HSTP→LSTP→SP

**信令分类**：按作用区域(用户线/局间)、按功能(监视/选择/音/维护)、按方向(前向/后向)、按传输方式(随路/共路)

### 三、路由算法（每年必考）

**路由算法两大功能**：
1. 为SD对选择传输路径
2. 选定路径后正确传送分组（通过路由表）

**数据报 vs 虚电路**：数据报每分组都重新选路；虚电路仅建立时选一次

**集中式 vs 分布式**（2024简答4）：集中式由RCC统一计算下发(全局最优但单点故障)；分布式各节点独立计算(鲁棒)

**广域网路由 vs 互联网路由**（2023简答3）：广域网=子网内(Dijkstra/距离矢量)；互联网=子网间(BGP)

**理想路由五大特点**(口诀"正简自稳公")：正确性、计算简单、自适应性(鲁棒)、稳定性(收敛无振荡)、公平性

**Dijkstra算法**(2025简答3)：逐步扩展已确定最短路径的节点集N'→选最小D(w)加入→更新邻居D(v)=min(D(v), D(w)+c(w,v))→循环至全节点。复杂度O(n²)

**距离矢量算法**(2025论述2)：Bellman-Ford方程 Dx(y)=min{c(x,v)+Dv(y)}；仅与邻居交换DV；存在计数到无穷问题→毒性逆转解决

**LS vs DV对比**：

| 维度 | LS(Dijkstra/OSPF) | DV(RIP/BGP) |
|------|-------------------|-------------|
| 交换范围 | 全网洪泛 | 仅邻居 |
| 收敛 | 快 | 慢(计数到无穷) |
| 代表协议 | OSPF/IS-IS | RIP/BGP |
| 扩展性 | 好(分层) | 差(15跳限制) |

**路由对网络性能影响**(2024论述1)：好/差路由可致通过量差3倍；高负荷下增加通过量，轻负荷下减少时延

### 四、QoS与宽带IP（IntServ/DiffServ）

**QoS定义**：服务性能总效果，决定用户满意度

**IP QoS五大模块**：标签→分类→管制→调度→呼叫接纳

**五种调度算法**：FIFO(简单/不公平)、严格优先级(低优先饿死)、WFQ(最好时延/IntServ用)、WRR(简单/无延时保证)、CBQ(分级)

**IntServ**(综合服务模型)——资源预留，每流绝对QoS：
- 三类服务：GS(硬保证)、CLS(受控负载/重载如轻载)、BE(尽力而为)
- 四部件：RSVP(信令/PATH→RESV，接收方控制)、接纳控制(判断是否接受新流)、分类器(五元组分类)、调度器(WFQ)
- 致命缺点：每流状态，扩展性差(状态爆炸)

**DiffServ**(区分服务模型)——边缘分类+核心简单转发：
- 三类业务：EF(DSCP=46, 静态QoS, 低延迟"虚拟专线")、AF(4等级×3丢弃=12码点, **动态QoS**)、BE(0)
- PHB：EF(加速转发)、AF(确保转发，有动态QoS——按阈值动态丢包)、BE
- 边界路由器TC五步：分类→计量(令牌桶)→标记(DSCP)→整形→丢弃
- **分类在边缘节点进行！**

**IntServ vs DiffServ对比**(核心)：

| 维度 | IntServ | DiffServ |
|------|---------|----------|
| 粒度 | 每流 | 每类 |
| 信令 | RSVP(有) | 无 |
| 扩展性 | 差 | 好 |
| QoS | 绝对(确定性) | 相对(统计性) |
| 场景 | 企业网/接入 | 运营商骨干 |

### 五、MPLS（三年连续考查）

**核心概念**(2.5层技术，路由控制与转发分离)：

| 术语 | 含义 |
|------|------|
| Label | 32bit标签(20Label+3EXP+1S+8TTL) |
| LER | 边缘路由器(PUSH入口/POP出口) |
| LSR | 核心标签交换路由器(SWAP) |
| FEC | 等价转发类，与Label一一对应 |
| LSP | 标签交换路径(单向) |
| LDP | 标签分发协议 |

**标签分配方式**：
- **下游分配**(2023简答6)：末端向上游分配；分DOD(按需)和DU(自主)；优点不产生重复
- **上游分配**(2024简答6)：起始向下游分配；上游自主决定通知下游

**MPLS工作流程**(2025论述5)：①LDP+IGP建立转发表和LSP→②入口LER识别FEC→PUSH标签→③核心LSR查标签表→SWAP转发→④出口LER→POP→IP转发

**MPLS解决的问题**：传统IP查表慢+缺流量工程+QoS难保证 → 标签交换快速转发+LSP显式路由+多协议兼容

**MPLS优点**：转发快、TE流量工程、VPN支持、QoS保障

### 六、智能网（三年均考）

**定义**：在原有通信网上叠加的附加网络，核心思想——**业务逻辑与交换功能分离**

**概念模型四平面**(2025简答7)：

| 平面 | 面向 | 核心内容 |
|------|------|---------|
| SP(业务) | 用户 | 25种业务、38种业务特征 |
| GFP(全局功能) | 业务设计者 | SIB+GSL+BCP |
| DFP(分布功能) | 网络设计者 | 9个FE(SCF是核心)+IF |
| PP(物理) | 实施者 | SCP/SSP/SCE/SMS/STP/IP |

**SIB**：标准化可重用功能构件；**GSL**：将SIB串接形成业务逻辑

**关键物理实体**(2024简答7)：SCP(业务控制核心/存储逻辑+数据)、SSP(智能呼叫检测触发)、IP(智能外设/语音/DTMF)、SCE(业务生成环境)、SMS(业务管理)

**INAP**：SSP↔SCP间高层信令协议，No.7/TCAP承载，ASN.1描述

**800业务流程**：用户拨800→SSP触发→InitialDP→SCP→翻译为真实号码→Connect→SSP接续

**智能网 vs 非智能网**：非智能=业务固化交换机(改代码/周期长/不灵活)；智能=业务与交换分离(SCP集中部署/开发快/开放)

### 七、NGN与软交换（三年均考）

**NGN定义**：以IP为中心的全业务分组网络，支持语音/数据/多媒体融合；不是革命而是**演进**

**四层架构**(2025简答8)：

| 层 | 功能 | 设备 | 协议 |
|----|------|------|------|
| 业务层 | 增值业务提供 | AS/SCP/AAA | SIP/Parlay/INAP |
| 控制层 | 呼叫控制/路由 | 软交换/MGC | H.248/SIP/BICC |
| 传送层 | IP骨干承载 | IP/MPLS路由器 | OSPF/BGP/MPLS |
| 接入层 | 异构接入适配 | MG/SG/IAD | H.248/SIGTRAN |

**软交换概念**(2023简答8)：呼叫控制从MG分离，纯软件实现。两个分离：**呼叫控制与承载分离、业务控制与呼叫控制分离**

**软交换六大功能**：媒体接入、呼叫控制、业务提供、互联互通、资源管理、认证计费

**关键设备**：MG(媒体格式转换PSTN↔IP)、SG(No.7↔SIGTRAN)、IAD(用户端综合接入，1-48端口)、AS(应用服务器)

**H.248 vs SIP对比**(2024简答8)：

| 维度 | H.248/Megaco | SIP |
|------|-------------|-----|
| 关系 | 主从(MGC→MG) | 对等(端到端) |
| 功能 | 网关控制 | 会话控制 |
| 编码 | ASN.1/Text | Text(类HTTP) |
| 场景 | 控制层↔接入层 | 业务层+终端 |

**SIP六种方法**：INVITE(发起)、ACK(确认)、BYE(释放)、CANCEL(取消)、REGISTER(注册)、OPTIONS(查询)

**SIP呼叫流程**(必画)：INVITE(SDP)→100 Trying→180 Ringing→200 OK(SDP)→ACK→RTP媒体流→BYE→200 OK

**SIP响应**：1xx临时(100/180)、2xx成功(200)、3xx重定向、4xx客户端错(404/486)、5xx服务端错、6xx全局错

**组网方案**：窄带(软交换替代PSTN长途/端局)、宽带(为IAD/智能终端提供IP多媒体)

### 八、卫星通信网（三年轮流考各分系统）

**四大分系统**：

| 分系统 | 作用 | 年份 |
|--------|------|------|
| 空间分系统 | 卫星本体，接收/放大/变频/转发信号 | 2024简答9 |
| 地球站分系统 | 发射/接收用户信号，中央站调度管理 | 2025简答9 |
| TT&C分系统 | 卫星跟踪/遥测/指令，轨道姿态控制 | — |
| 监控管理分系统 | 参数测试鉴定、业务监视管理、资源分配 | 2023简答9 |

**轨道分类**：GEO(35786km, 270ms延迟)、MEO(2000-20000km)、LEO(200-2000km, ~25ms)

**频段**(从低到高)：L(1-2G/导航)→S(2-4G/移动)→C(4-8G/固定/雨衰小)→Ku(12-18G/直播/雨衰)→Ka(18-40G/高通量/大带宽雨衰严重)

**转发器两种**：透明转发器(放大+变频/简单/噪声累积)、再生转发器(解调+处理+再调制/噪声不累积/复杂昂贵)

**卫星组成**：空间平台(服务舱：结构/温控/电源/姿控/遥测指令) + 有效载荷(天线+转发器)

**七大优点**：距离远费用低、广播多址、容量大、自发自收监测、无缝覆盖、广域组网、可靠(应急)
**三大缺点**：时延长(GEO~270ms)、链路受外部影响(雨衰)、星蚀和日凌中断

**中国卫星时事素材**：北斗(BDS/GEO+IGSO+MEO/独有短报文)、鹊桥(L2 Halo轨道/月背中继)、千帆星座(1.2万颗LEO)、星网(SatNet)、鸿雁/虹云

### 九、网络管理与TMN（三年均考）

**FCAPS五大功能**(必背)：

| 字母 | 功能 | 管理内容 |
|------|------|---------|
| F | 故障管理 | 差错日志/告警/定位隔离/诊断/排除 |
| C | 配置管理 | 初始化/删除/设参数/收集状态/拓扑/升级 ← 2023简答10 |
| A | 计费管理 | 成本核算/收费/阈值/统计 |
| P | 性能管理 | 统计数据/历史记录/模拟/容量规划 |
| S | 安全管理 | 安全服务/日志/分发/接入控制/认证/密钥 |

**TMN定义**：独立于电信网的标准化管理网络，通过标准接口交换管理信息

**TMN功能体系结构**(2023论述2/2024论述5)：
- 六大功能块：OSF(执行FCAPS管理)、NEF(网元抽象)、WSF(人机界面f→g)、MF(协议转换/汇聚/过滤)、QAF(TMN↔非TMN适配)、DCF(数据传输贯穿全块)
- 参考点：**q3**(OSF↔MF/NEF/OSF，最重要全功能)、qx(局部管理)、f(工作站)、x(跨TMN域)、g(人机)、m(非TMN适配)
- 对应接口：Q3(最重要，协议栈+管理协议+信息模型)、Qx、F、X

**TMN物理体系结构**(2025简答10)：六大物理元素——OS(运营系统)、MD(中介设备)、QA(Q适配器)、DCN(数据通信网)、NE(网元)、WS(工作站)

**SNMP模型**：Manager(管理站)→Get/Set→Agent(代理)；Agent主动发Trap告警。通过UDP 161/162端口操作

**SNMP vs CMIP**：SNMP简单/Internet广泛/TCP-IP；CMIP复杂/电信领域/OSI

**两大管理协议**：SNMP(IETF/TCP-IP/简单)、CMIP(ISO/OSI/复杂)
