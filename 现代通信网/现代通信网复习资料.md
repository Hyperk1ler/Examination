## 二、第一章 绪论——现代通信网概论

### 2.1 知识框架

```
现代通信网概论
├── 通信基本模型：信源→变换器→信道→反变换器→信宿（+噪声源）
├── 通信网构成
│   ├── 硬件：终端设备、传输系统、转接交换系统
│   └── 软件：信令方案、协议、网络结构、路由方案、编号方案、资费制度、质量标准
├── 通信网五大特征
│   ├── 多样化：电话网、互联网、传感器网、3G/4G/5G/5G-A并存
│   ├── 宽带化：5G峰值速率几十Gb/s，时延降到几ms，支持100万用户/km²
│   ├── 移动化：移动用户已超固定用户
│   ├── 泛在化：网络无处不在，人-人、人-机、机-机通信
│   └── 可信化：行为可监测、可评估、异常可管理
├── 七种分类方式
│   ├── 按业务类型、服务区域、传输媒介、交换方式、结构、信息信号形式、信息传递方式
├── 拓扑结构
│   ├── 总线型、星状、环状、树状、网状（各自的优缺点）
├── 传输方式
│   ├── 广播式（需MAC协议，如总线型/环型）
│   └── 点到点式（需路由选择，如星型/树型/网状）
├── 交换技术
│   ├── 电路交换：建立→传输→拆除，面向连接，时延小，利用率低
│   ├── 分组交换：存储转发，高效灵活，时延抖动
│   ├── ATM：53字节定长信元（5B信头+48B信息），面向连接，QoS好但复杂
│   └── IP交换：TCP/IP统一协调，结合L2交换+L3路由优势
├── 复用技术
│   ├── TDM（同步/异步）、FDM、WDM、CDM/CDMA
├── 功能结构（三大网）
│   ├── 业务网：提供语音、数据、图像、多媒体等通信业务
│   ├── 传送网：提供透明传输通道（PDH→SDH→OTN）
│   └── 支撑网：信令网 + 同步网 + 管理网
└── 发展趋势
    ├── 网络融合（话音/数据、电路/分组、电/光、有线/无线）
    ├── NGN：以IP为中心的全业务网络
    └── 空天地海一体化网络
```

### 2.2 核心概念

#### 🔑 通信网的定义
由一定数量的**节点**（终端节点和交换节点）和**传输链路**有机组合，实现两个或多个规定点间信息传递的通信体系。

#### 🔑 现代通信网的构成

| 层级 | 组成 | 说明 |
|------|------|------|
| **硬件** | 终端设备 | 信源/信宿，如电话、电脑 |
| | 传输系统 | 信道+变换器+反变换器 |
| | 转接交换系统 | 交换节点，实现路由选择 |
| **软件** | 信令方案、协议、网络结构、路由方案、编号方案、资费制度、质量标准 | 保证网络有序运行 |

#### 🔑 五大特征详解

| 特征 | 含义 |
|------|------|
| **多样化** | 多种网络制式并存（PSTN、Internet、传感器网、3G/4G/5G/5G-A） |
| **宽带化** | 传输速率极大提升，5G峰值数十Gb/s，时延降至毫秒级 |
| **移动化** | 移动通信用户数超过固定用户 |
| **泛在化** | 网络覆盖无处不在，实现人-人、人-机、机-机全连接（RFID+传感器） |
| **可信化** | 网络行为可监测、可评估、异常可管理；支持网络共存、互联与融合 |

### 2.3 交换技术对比

| 交换方式 | 原理 | 面向连接 | 优点 | 缺点 |
|----------|------|---------|------|------|
| **电路交换** | 建立电路→传输→拆除 | ✅ | 时延小、无抖动、实时性好 | 电路利用率低 |
| **分组交换** | 存储转发 | ❌ | 高效灵活、资源共享 | 时延抖动、无序到达 |
| **ATM交换** | 53字节定长信元交换 | ✅ | 高速、QoS保证 | 技术复杂、开销大 |
| **IP交换** | TCP/IP协议统一协调 | ❌ | 屏蔽硬件细节、L2+L3优势 | 尽力而为、无QoS保证 |

### 2.4 复用技术

| 复用方式 | 原理 | 关键点 |
|----------|------|--------|
| **TDM（时分复用）** | 时间片划分 | 同步TDM（固定分配） vs 异步TDM（按需分配） |
| **FDM（频分复用）** | 频率划分 | 不同信号调制到不同频段 |
| **WDM（波分复用）** | 波长划分 | 光纤通信中使用，DWDM扩展带宽 |
| **CDM（码分复用）** | 码片序列正交 | m bit码片实现扩频，CDMA |

### 2.5 传送网技术演进

```
PDH（准同步数字体系）
  ├── E系列：2.048 Mbps
  ├── T系列：1.544 Mbps
  └── 缺点：地区性标准、无标准光接口、逐级复接不灵活、管理能力弱
        ↓
SDH（同步数字体系）
  ├── 标准光接口、全球统一速率等级
  ├── 155M / 622M / 2.5G / 10G / 40G
  └── 优点：统一标准、强大OAM能力、灵活上下电路
        ↓
OTN（光传送网络）
  ├── 综合SDH优点 + DWDM带宽可扩展性
  └── 下一代传送网核心技术
```

### 2.6 支撑网详解

#### 信令网
- **信令分类**：
  - 按作用区域：用户线信令 / 局间信令
  - 按功能：监视信令 / 选择信令 / 音信令 / 维护管理信令
  - 按传输方式：随路信令 / 共路信令（No.7信令）
- **No.7信令系统四级结构**：
  - MTP-1（信令数据链路功能）：物理层
  - MTP-2（信令链路功能）：差错控制、流量控制
  - MTP-3（信令网功能）：消息路由、信令网管理
  - UP（用户部分）：TUP/ISUP/SCCP/TCAP等
- **三种信令单元**：MSU（消息信令单元）、LSSU（链路状态信令单元）、FISU（填充信令单元）
- **信令网组成**：SP（信令点）、STP（信令转接点）、信令链路
- **信令网结构**：无极信令网 / 分级信令网

#### 同步网
- 在通信设备之间提供**基准定时信号**，保证数字通信网同步运行

#### 管理网
- 监测和控制通信网络运行状态（详见第五章）

```

│   ├── 网络综合演进：PSTN→ISDN→ATM(失败)→IP(成功)
│   ├── QoS保证
│   │   ├── QoS指标：可用性、延迟、抖动、吞吐量、丢包率

    │   ├── 业务平面(SP)：25种业务、38种业务特征

    │   ├── 分布功能平面(DFP)：9个功能实体(FE)
    │   └── 物理平面(PP)：SCP、SSP、STP、IP、SMS、SCE
    ├── 主要物理实体
    │   ├── SCP（业务控制点）：智能网核心，存储用户数据和业务逻辑

    │   ├── IP（智能外设）：语音合成、播放录音、DTMF接收、语音识别

    │   └── SCE（业务生成环境）：图形化业务设计平台

### 3.2 QoS核心内容

#### QoS五大指标

| 指标 | 含义 |
|------|------|
| **可用性** | 用户使用服务的能力（网络正常运行时间比例） |
| **延迟（时延）** | 数据从源到目的地所需时间 |
| **延迟抖动（Jitter）** | 时延的变化幅度 |
| **吞吐量** | 单位时间内成功传输的数据量 |
| **丢包率** | 传输过程中丢失数据包的比例 |

#### 调度机制对比

| 调度算法 | 特点 | 适用场景 |
|----------|------|---------|
| **FIFO** | 先到先服务，不能区分优先级 | 简单网络 |
| **严格优先级** | 高优先级先处理 | 实时业务保障 |
| **WFQ（加权公平排队）** | 时延性能最好，但复杂度高 | 需要精确QoS |
| **WRR（加权轮转）** | 实现简单，不提供延时保证 | 通用场景 |
| **CBQ（基于级别排队）** | 树状分级结构 | 多层次QoS |

### 3.3 IntServ vs DiffServ 对比 ⭐（高频考点）

| 对比维度 | IntServ（综合服务） | DiffServ（区分服务） |
|----------|-------------------|---------------------|
| **基本思想** | 端到端资源预留 | 边界分类+核心简单转发 |
| **QoS保证** | 绝对有保证（per-flow） | 相对有保证（per-class） |
| **可扩展性** | 差（状态信息与流数成正比） | 好（流聚合） |
| **复杂度** | 高（信令+状态维护） | 低（边界复杂，核心简单） |
| **三类服务** | 有保证服务、受控负载服务、尽力而为 | 优质业务(PS)、确保业务(AS)、尽力而为(BE) |
| **核心协议/机制** | RSVP（资源预留协议） | DSCP标记 + PHB（每跳行为） |
| **功能部件** | RSVP、接纳控制、分类器、调度器 | 分类器、通信量调节器（测量/标记/整形/丢弃） |

### 3.4 MPLS 详解

#### 标签结构（共32位）
```
┌──────────────┬──────┬───┬──────┐
│  Label(20b)  │COS(3b)│S(1b)│TTL(8b)│
└──────────────┴──────┴───┴──────┘
```

#### 关键术语

| 术语 | 全称 | 含义 |
|------|------|------|
| **LSR** | 标签交换路由器 | MPLS网络核心节点，基于标签转发 |
| **LER** | 边缘标签路由器 | MPLS网络入口/出口节点 |
| **FEC** | 等价转发类 | 以相同方式转发的一组IP分组 |
| **LSP** | 标签交换路径 | 从入口LER到出口LER的路径 |
| **LDP** | 标签分配协议 | 在LSR之间分发标签 |

#### 标签分配方式

| 分配方式 | 发起方 | 方向 |
|----------|-------|------|
| **下游分配** | 下游LSR | 下游→上游（请求方式/主动方式） |
| **上游分配** | 上游LSR | 上游→下游 |

#### MPLS优缺点

- **优点**：转发简单、FEC增强可扩展性、支持QoS路由和流量工程
- **缺点**：短信令开销大、IP分组流自动识别待提高

### 3.5 智能网(IN)

#### 智能网的核心思想
**业务逻辑与基础通信功能分离**——交换层处理基本呼叫连接，智能层负责增值业务控制。

#### INCM四平面模型

```
业务平面(SP) ──用户视角──→ 25种业务、38种业务特征
      ↓
全局功能平面(GFP) ──设计者视角──→ SIB（业务独立构件）+ GSL（全局业务逻辑）
      ↓
分布功能平面(DFP) ──网络视角──→ 9个功能实体(FE)、FEA、信息流(IF)
      ↓
物理平面(PP) ──实现视角──→ 物理实体：SCP、SSP、STP、IP、SMS、SCE
```

#### 主要物理实体及功能

| 实体 | 功能 |
|------|------|
| **SCP**（业务控制点） | 智能网核心，存储用户数据和业务逻辑，双/三备份 |
| **SSP**（业务交换点） | 以程控交换机为基础 + No.7信令接口，触发智能业务 |
| **STP**（信令转接点） | No.7信令网转接点 |
| **IP**（智能外设） | 语音合成、播放录音、DTMF接收、语音识别 |
| **SMS**（业务管理系统） | 业务逻辑/数据/用户数据管理、监测、业务量管理 |
| **SCE**（业务生成环境） | 图形化软件设计平台，快速开发新业务 |

#### 典型的800业务（FPH）处理流程

```
用户拨800号码 → 交换机送SSP → SSP查询SCP → SCP译码表翻译成普通号码
→ 回送SSP → SSP完成电路连接
```

## 四、第三章 软交换和下一代网络

### 4.1 知识框架

```
第三章 软交换和下一代网络
├── NGN（下一代网络）
│   ├── 产生背景：Internet冲击 → 电信效益滑坡 → NGI/Internet2 → 软交换
│   ├── ITU-T定义(2004)：基于分组的全业务网络
│   ├── 主要特征：分组传送、三分离（控制/承载、呼叫/会话、业务/网络）、开放接口

### 4.2 NGN核心内容

#### NGN定义（ITU-T, 2004年）
> NGN是一个**分组网络**，提供包括电信业务在内的多种业务，利用多种带宽和具有QoS能力的传送技术，实现**业务功能与底层传送技术的分离**，支持通用移动性。

#### NGN主要特征

1. **基于分组传送**（核心）
2. **控制与承载分离、呼叫与会话分离、业务与网络分离**（三分离原则）
3. 开放接口
4. 宽带能力
5. 与传统网络互通
6. 通用移动性
7. 融合固定与移动业务

#### NGN功能分层（ITU-T）

| 层次 | 功能 |
|------|------|
| **接入层** | 各类用户终端连接至网络，提供各种接入方式 |
| **传送层** | 信息格式转换和媒体流选路 |
| **控制层** | 呼叫控制功能（核心层） |
| **业务层** | 提供增值业务，第三方可编程接口 |

### 4.3 软交换详解

#### 软交换概念
将**呼叫控制功能**从媒体网关（传输层）中分离出来，通过**服务器软件**实现基本呼叫控制功能（呼叫选路、管理控制、连接控制、信令互通）。

#### 分层优点

| 分离方式 | 好处 |
|----------|------|
| 业务提供与网络控制分离 | 快速提供新业务 |
| 呼叫控制与承载分离 | 便于采用新传送技术 |
| 承载与接入分离 | 便于各种现有网络接入 |
| 模块化采购 | 可从不同制造商购买最合适部件 |

### 5.4 卫星通信优缺点

4. **可自发自收监测**
5. **无缝覆盖能力**
6. **广域复杂网络拓扑构成能力**
7. **安全可靠性**：抗震救灾、光缆故障时的备份

1. **通信时延长**：同步卫星约270ms（单向）
2. **链路易受外部条件影响**：雨衰（Ka频段严重）、电离层闪烁等

> **星蚀**：卫星进入地球阴影区，太阳能电池无法供电

## 七、重点专题：路由算法

> 📌 **说明**：本部分内容来自老师在课程后期补充发放的两份专题资料（1.pdf「什么是路由算法？」和2.pdf「路由算法的实现？」），由西安电子科技大学通信工程学院刘勤编写，是考试重点。

### 7.1 路由算法的核心地位

路由算法是**网络层的核心问题**。网络层负责通过中间节点连接两个端系统，形成通信路径，实现**透明传输**（任何信息以原样送达，网络不修改内容）。

路由的两大核心功能：

| 功能 | 说明 |
|------|------|
| **① 路径选择** | 为每个源-目的(SD)对选择一条传输路径 |
| **② 分组转发** | 路径选定后，正确地将用户报文送达目的地（使用**路由表**） |

### 7.2 数据报方式 vs 虚电路方式

| 对比维度 | 数据报方式 | 虚电路方式 |
|----------|----------|----------|
| **路由决策时机** | 每个分组独立决策 | 建立虚电路时一次性决策 |
| **路径一致性** | 各分组可能走不同路径 | 同一条虚电路的分组走相同路径 |
| **连接建立** | 无连接 | 面向连接（需建立/拆除） |
| **灵活性** | 高，可动态适应网络变化 | 低，但可保证顺序和QoS |

### 7.3 λ₁/λ₂ 经典路由例题 ⭐

**网络拓扑**：
- 节点1、2为源节点，节点6为目的节点
- 节点1连接节点3、4；节点2连接节点4、5
- 节点3、4、5均连接目的节点6
- **所有链路容量 = 10单位**

#### 情况1：λ₁ = 5, λ₂ = 5（各源节点5单位流量）

| 方案 | 路径选择 | 链路负载 | 效果 |
|------|---------|---------|------|
| **好路由** | 1→3→6，2→5→6 | 每条链路50%容量 | 时延小 ✅ |
| **坏路由** | 都走节点4→6 | 4-6链路100%容量 | 时延大 ❌ |

#### 情况2：λ₁ = 5, λ₂ = 15

- 节点2有15单位流量，但单链路容量仅10 → 必须丢包或分流
- **好方案（负载分担）**：节点2分流至 2→4→6 和 2→5→6，节点1走1→3→6
- 结果：无链路超过75%容量，时延较小

#### 核心结论
> 路由算法选择不同，网络最大吞吐量可在10~30单位之间变化。好的路由算法应该：**高负载下提高吞吐量，低/中负载下降低每分组平均时延**。

### 7.4 路由算法四大设计问题

1. **用什么策略**选择路由？
2. **基于什么信息**进行选择？
3. **如何执行**路由选择策略？
4. **用什么准则**评价路径好坏？

### 7.5 路由算法分类体系 ⭐

| 分类维度 | 具体类别 |
|----------|---------|
| **决策地点** | 每节点决策；中心节点决策；源节点决策；节点子集决策 |
| **决策时间** | 每分组（数据报）；每会话（虚电路） |
| **性能准则** | 链路数最少；设施成本最低；时延最小；吞吐量最大 |
| **网络信息源** | 无信息；本地信息；相邻节点信息；路径上节点信息；全网信息 |
| **路由策略** | 静态（简单算法）；自适应（连续更新/周期性更新/负载剧变更新/拓扑变化更新） |

### 7.6 三大分类维度详解

#### (1) 非自适应（静态路由）vs 自适应

| 类型 | 特点 | 优点 | 缺点 |
|------|------|------|------|
| **非自适应（静态）** | 路由预先算好，启动时下载 | 简单、开销小 | 不根据实时条件调整 |
| **自适应** | 根据测量/估计的当前条件调整 | 对网络变化反应快 | 复杂、开销大 |

#### (2) 集中式 vs 分布式

| 类型 | 工作方式 | 优点 | 缺点 |
|------|---------|------|------|
| **集中式路由** | 路由控制中心(RCC)周期性收集全网链路状态，计算路由，分发路由表 | 全局最优决策 | 单点故障、扩展性差 |
| **分布式路由** | 所有节点相互交换路由信息，各自独立计算路由 | 健壮、可扩展 | 可能不一致/振荡 |

#### (3) 广域网路由 vs 互联网路由

| 类型 | 解决的问题 | 常用算法 |
|------|----------|---------|
| **广域网路由** | 单一子网内部的路由 | Dijkstra最短路径、Flooding、距离向量等 |
| **互联网路由** | 不同子网之间的路由 | BGP、OSPF、RIP等 |

### 7.7 路由表

每个节点维护的**路由表**记录：

| 字段 | 说明 |
|------|------|
| 目的地址 | 分组要到达的目标节点 |
| 下一跳（输出链路） | 应转发到哪个相邻节点/接口 |
| 路径质量 | 该路由的时延/带宽/代价等度量 |
| 利用率 | 该路由当前的负载情况 |

路由表根据路由算法规则动态修改和更新。

### 7.8 理想路由算法的五个特性 ⭐⭐⭐

| 特性 | 含义 | 具体要求 |
|------|------|---------|
| **① 正确性** | 分组必须准确到达正确目的地 | 按路由表方向转发，不得转发越过目的地 |
| **② 计算简单** | 最小化节点资源消耗 | 减少延迟、减少链路带宽用于路由开销、不要大量收集其他节点状态信息 |
| **③ 自适应性/鲁棒性** | 适应流量和拓扑变化 | 节点/链路故障时快速找到替代路径；故障恢复后迅速利用 |
| **④ 稳定性/收敛性** | 不能过度振荡 | "振荡"=路径在多个选项中不停切换而不稳定；必须收敛到稳定状态 |
| **⑤ 公平性** | 公平对待所有用户 | 不能为优化某对用户端到端时延而使其占用不成比例的网络资源 |

> ⚠️ **关键结论**：没有单一算法能同时满足所有五个特性（有些甚至互相矛盾，如最大化吞吐量可能增大时延）。好的路由算法应在吞吐量阈值以下保持低时延；超过阈值时通过**流量控制**使网络运行在阈值以下。
