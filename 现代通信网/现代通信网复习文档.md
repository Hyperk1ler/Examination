## 四、各题型答题技巧

### 简答题（5分/题，建议每题约8分钟）

- **第一步**：先写核心定义（1-2句），开门见山

- **第三步**：必要时举例说明，增强说服力
- **关键原则**：踩分点要突出，可使用编号或分段使层次清晰

### 论述题（10分/题，建议每题约20分钟）

- **采用四段式结构**：
  1. **背景介绍**：点明题目涉及的技术/概念产生的背景与需求

  3. **优势挑战/原理展开**：分析该技术的优势、局限性或核心原理
  4. **意义展望**：总结该技术的意义并适当展望发展趋势

### 画图题

- **步骤一**：先画主体框架（核心模块/层级关系）
- **步骤二**：标注关键组件和接口/协议名称
- **步骤三**：加文字说明，补充图中未体现的关键信息
- **实用建议**：用铅笔先勾勒轮廓，确认无误后再描实

# 第二部分：第一章 通信网络基础

## 1.1 通信基本模型

### 六大要素

```
信源 → 变换器 → 信道 → 反变换器 → 信宿
                  ↑
              噪声源（作用于整个通信过程）
```

| 要素 | 定义 | 说明 |
|------|------|------|
| **信源** | 发出信息的基本设施 | 如电话机、计算机、摄像头等终端设备 |
| **变换器** | 对信源信息按一定目的进行变换的设备 | 如调制器、编码器等，将原始信号转换为适合信道传输的形式 |
| **信道** | 信息传输介质的总称 | 按传输介质分：有线信道和无线信道；按传输信号形式分：模拟信道和数字信道 |
| **反变换器** | 变换器的逆工作过程 | 如解调器、解码器等，将信道传输的信号还原为原始信息 |
| **信宿** | 信息传输的终点，信息的接收者 | 信息最终到达的目的设备 |
| **噪声源** | 不是人为实体，但通信过程中实际存在 | 对信号传输产生干扰的各种因素的总称 |

### 现代通信网络定义

由终端、传输和交换等现代通信设备组成的系统，用以实现语音、数据、多媒体形式的通信要求。

### 现代通信网构成

- **硬件（三部分）**：
  1. 终端设备
  2. 传输系统
  3. 转接交换系统

- **软件**：信令方案、各种协议、网络结构、路由方案、编号方案、资费制度与质量标准

## 1.2 通信网络五大特征（2024简答1！）

| 特征 | 核心含义 | 具体体现 |
|------|---------|---------|
| **多样化** | 存在形式多样，网络结构各层面多样 | 电话网、互联网、传感器网、3G/4G/5G网等多种网络并存；接入层、汇聚层、核心层各具特点 |
| **宽带化** | 业务宽带化与干线宽带化并行推进 | 用户接入速率从28.8kbps发展到100Mbps/Gbps；5G峰值速率达几十Gb/s，端到端延时从4G十几ms减少到几ms；支持100万用户/平方公里 |
| **移动化** | 移动通信成为主流接入方式 | 移动用户数量已超过固定电话用户数，移动互联网成为主要上网方式 |
| **泛在化** | 网络无处不在，主动感知用户需求 | RFID+传感器构成无处不在网络；引发自律网研究（自组织/自愈/自管理/自优化） |
| **可信化** | 网络和用户行为结果可预期与可管理 | 属性包括安全性、可生存性、可管理性；含服务提供者可信、网络信息传输可信、终端用户可信三个层面 |

## 1.3 通信网络七种分类方式

| 分类维度 | 类型举例 |
|---------|---------|
| 按业务类型 | 电话网、电报网、数据通信网、传真通信网、图像通信网、有线电视网 |
| 按服务区域 | 本地电信网、农村电信网、长途电信网、移动通信网、国际电信网 |
| 按传输媒介 | 架空明线网、电缆通信网、光缆通信网、卫星通信网、低轨道卫星移动通信网 |
| 按交换方式 | 电路交换网、报文交换网、分组交换网、宽带交换网 |
| 按网络结构 | 网状网、星形网、环形网、栅格网、总线网 |
| 按信息信号形式 | 模拟通信网、数字通信网、数字/模拟混合网 |
| 按信息传递方式 | 同步转移模式（STM）网络、异步转移模式（ATM）网络 |

## 1.4 网络拓扑结构（五种）

| 拓扑 | 结构特点 | 优点 | 缺点 |
|------|---------|------|------|
| **总线型** | 所有节点连接到一条共享总线，以广播方式传输 | 电缆短、布线方便、易于扩充 | 节点故障可能致全网瘫痪、故障定位困难 |
| **星状** | 中央节点通过点到点链路连接各周边节点 | 建网灵活、单个节点故障不扩散 | 严重依赖中央节点，中央节点故障全网瘫痪 |
| **环状** | 节点首尾相连形成闭合环路，类似总线型首尾相接 | 适合光纤传输（如FDDI），结构简单 | 对环路可靠性依赖强，单点故障可致全网中断 |
| **树状** | 倒置树形的分层结构，从根节点向下逐级扩展 | 有一定容错能力，适合分级管理 | 依赖上层节点，根节点故障影响面大 |
| **网状** | 任意两节点之间存在多条可达链路可选 | 可靠性最高，多路径容错 | 存在冗余链路，传输效率相对较低，成本高 |

## 1.5 两种传输方式

| 对比维度 | 广播式 | 点到点式 |
|---------|--------|---------|
| **原理** | 所有节点共享同一传输信道 | 一条传输路径连接两台设备，需经多台设备存储转发 |
| **关键技术** | 介质访问控制（MAC）方法 | 路由选择（Routing） |
| **典型拓扑** | 总线型、环状 | 星型、环型、树型、网状 |
| **代表实例** | 共享式以太网、WLAN的DCF模式 | 广域网、交换式以太网、WLAN的PCF模式 |

## 1.6 交换技术（重点！）

### 四种交换技术对比表（必背！）

| 特征 | 电路交换 | 分组交换 | ATM | IP交换 |
|------|----------|----------|-----|--------|
| 连接方式 | 面向连接 | 无连接/虚电路 | 面向连接 | 无连接 |
| 复用方式 | TDM | 统计复用 | 统计复用（信元） | 统计复用 |
| 交换粒度 | 时隙 | 分组（变长） | 信元（53字节，定长） | 分组（变长） |
| 时延 | 固定、小、无抖动 | 可变、有抖动 | 低、可预测 | 尽力而为 |
| 资源效率 | 低（独占资源） | 高 | 中 | 最高 |
| QoS保障 | 最好（专用通路） | 无 | 好 | 需额外提供（如DiffServ/IntServ） |
| 典型应用 | 电话网、ISDN | ARPANET、互联网 | 骨干网（现已较少使用） | 现代互联网 |

### 电路交换

- **三个阶段**：建立电路 → 传输数据 → 拆除电路
- **分类**：空分交换（不同空间路径）和时分交换（不同时隙）
- **特点**：
  - 优点：实时性强、时延小且无抖动、传输透明、不需存储分析处理、效率高
  - 缺点：建立时间长、电路利用率低（通话间隙资源闲置）
- **典型应用**：电话网、ISDN

### 分组交换（2025简答4！）

- **基本原理**：**存储转发**——将用户数据分割成若干分组，每个分组携带地址信息独立选路转发
- **世界上第一个分组交换网**：ARPANET（1969年）
- **优势**：
  - 高效：统计复用，线路利用率高
  - 灵活：每个分组独立选路，自适应网络变化
  - 迅速：不需要预先建立端到端连接
  - 可靠：逐段差错控制和流量控制
- **劣势**：
  - 传输时延和抖动不适合实时业务
  - 额外开销（每个分组需携带分组头信息）
  - 管理和控制较复杂
- **与电路交换的根本区别**：电路交换先建立连接再传输，独占资源；分组交换采用存储转发，统计复用共享资源

### ATM（异步传递方式）

- ITU-T于1988年正式提出，作为B-ISDN的核心技术
- **信元（Cell）** = 53字节（5字节信头 + 48字节信息段），**定长分组**
- **与STM（同步传递方式）的区别**：ATM各路信号不按固定时间间隔出现，需附加标志标明所属虚连接
- **优点**：高速率、高吞吐率、低时延、QoS保证（面向连接+定长信元）
- **缺点**：技术复杂、桌面业务开发欠缺、实现过于复杂，已被IP技术逐步取代

### IP交换

- **核心思想**：通过TCP/IP协议抽象和屏蔽硬件细节，建立统一协调的通信系统
- 将第2层交换机（高速转发）和第3层路由器（智能选路）的优势结合
- **特点**：以1/10的代价获得近10倍的性能提升

## 1.7 信道复用技术（四种）

| 复用方式 | 原理 | 关键特点 |
|---------|------|---------|
| **TDM（时分复用）** | 将时间分成等长时隙，以轮转方式分配给各路信号 | 分同步TDM（固定分配）和异步TDM/统计时分复用（按需动态分配） |
| **FDM（频分复用）** | 将不同信号调制到不同频率范围，然后合成一路传输 | 需设置保护频带防止相邻信道干扰；各信道同时占用不同频段 |
| **WDM（波分复用）** | 将整个波长频带划分为若干波长范围，每路信号使用不同波长 | CWDM（粗波分，20nm间隔）/ DWDM（密集波分，0.8nm间隔）；三个波长窗口：850nm、1310nm、1550nm；光纤带宽超50Tbps（实验室） |
| **CDM/CDMA（码分复用）** | 不同用户使用不同的正交码型，在同一频率同一时间传输 | 需满足正交关系：S·T=0（不同码正交），S·S=1（自相关为1）；实现扩频通信，抗干扰能力强 |

## 1.8 传送网

### 传送网定义

为交换节点/业务节点之间提供信息透明传输通道的网络，独立于业务网。

### 通信网三层功能结构

```
        ┌──────────────────────────┐
        │       业务网              │  ← 向用户提供语音、数据、图像、多媒体、VPN等通信业务
        ├──────────────────────────┤
        │       传送网              │  ← 提供透明传输通道（本节内容）
        ├──────────────────────────┤
        │       支撑网              │  ← 保障业务网正常运行、增强网络功能、提高服务质量
        └──────────────────────────┘
```

- **业务网**：面向用户，提供各种通信业务
- **传送网**：为上层业务提供透明传输通道，是通信网的"骨架"
- **支撑网**：为业务网提供信令、同步、管理支撑（见1.9节）

### 传输介质

#### 有线介质

| 介质类型 | 特点 | 典型应用 |
|---------|------|---------|
| **双绞线（STP/UTP）** | 价格最低，安装方便 | 100BaseT以太网，最大传输距离100m |
| **同轴电缆（75Ω/50Ω）** | 75Ω适合传输模拟信号（达几十km）；50Ω适合传输数字信号（达几km） | 有线电视（75Ω）、早期以太网（50Ω） |
| **光纤（单模/多模）** | 三个波长窗口：850nm / 1310nm / 1550nm；带宽超50Tbps（实验室） | 长途干线、城域网、FTTH接入 |

#### 无线介质

- 无线电波（WiFi、蓝牙、移动通信）
- 红外线（遥控器、短距离通信）
- 激光（激光通信，需视距传输）

### 技术体制演进：PDH → SDH → OTN

#### PDH（准同步数字体系）

- **两类系列**：
  - 欧洲系列（中国采用）：E1 = 2.048Mbps，30/32路PCM
  - 日美系列：T1 = 1.544Mbps，24路PCM
- **速率等级（欧洲系列）**：
  - E1 = 2.048Mbps
  - E2 = 8.448Mbps（4路E1复用）
  - E3 = 34.368Mbps（4路E2复用）
  - E4 = 139.264Mbps（4路E3复用）
- **四大缺点**（口诀"缺准管难"）：
  1. **缺**乏世界性标准光接口（不同厂家设备无法光口互联）
  2. **准**同步导致滑码（各节点时钟独立，存在频率偏差）
  3. **管**理控制比特少（OAM能力不足）
  4. 逐级复接不灵活，上下电路困**难**（需逐级复用/解复用）

#### SDH（同步数字体系）

- **速率体系**：
  - STM-1 = 155.52Mbps
  - STM-4 = 622Mbps
  - STM-16 = 2.5Gbps
  - STM-64 = 10Gbps
  - STM-256 = 40Gbps
- **特点**：
  - 统一标准光接口（不同厂家设备可互联互通）
  - 丰富的管理开销字节，便于维护管理
  - 灵活上下电路（分插复用器ADM）

#### OTN（光传送网）

- 综合了SDH的运维管理优势和DWDM的带宽可扩展性
- 是当前骨干传送网的主流技术

## 1.9 支撑网

### 支撑网定义（作业总结）

支撑网是不直接传送用户业务，而是为业务网的正常运行提供**控制**、**管理**和**同步**功能的背后网络。

### 三大支撑网

| 支撑网 | 功能定位 | 核心内容 |
|--------|---------|---------|
| **信令网** | 通信网的"神经系统" | 传递控制信息，协调交换设备工作；详见1.11节No.7信令系统 |
| **同步网** | 通信网的"时钟系统" | 在通信设备间提供基准定时信号，实现时钟信号速率同步；由节点时钟设备和定时分配链路组成 |
| **管理网** | 通信网的"管理大脑" | 监测和控制网络运行状态；详见第五章TMN |

## 1.10 数据传输链路（2025简答1！）

### 两类链路的区别

| 链路类型 | 含义 | 特征 |
|---------|------|------|
| **物理传输链路接通** | 能传送比特流 | 不可靠（无检测和纠错机制） |
| **数据传输链路接通** | 具备检测、确认和重传功能 | 实现可靠传输（如HDLC、PPP等数据链路层协议） |

- **关键区别**：物理传输链路只能实现"比特流能传过去"；数据传输链路在此基础上增加了差错控制机制，确保数据**可靠**传输。
- 两类链路的概念层次：物理传输链路（物理层）+ 数据链路控制层 = 数据传输链路

## 1.11 No.7信令系统（重点！）

### 信令基本概念

**信令定义**：终端和交换机之间、交换机之间传递的控制信息，是通信网的**神经系统**，控制着通信过程的建立、维持和释放。

### 信令分类

| 分类维度 | 类型 | 说明 |

| | **局间信令** | 交换机与交换机之间传递的信令 |

| | **维护管理信令** | 用于网络的维护、管理和故障处理 |
| 按传送方向 | **前向信令** | 主叫方向 → 被叫方向 |
| | **后向信令** | 被叫方向 → 主叫方向 |
| 按传输方式 | **随路信令** | 信令与话音共用同一物理通道（一个话路配一个信令通道），效率低 |
| | **共路信令** | 使用专用高速数据链路传送大量话路的信令，No.7信令属于此类 |

| 类型 | 全称 | 用途 | 特征 |

| **LSSU** | 链路状态信令单元（Link Status Signal Unit） | 传送链路状态信息（如链路拥塞、链路故障） | 长度较短，仅含状态字段 |

### 信令网组成与结构

#### 三个基本实体

| 实体 | 全称 | 功能 |

| **STP** | 信令转接点（Signaling Transfer Point） | 转接信令消息，分为独立STP和综合STP（与交换功能合设） |
| **信令链路** | Signaling Link | 连接SP和STP的物理链路，负责传送信令消息 |

- **无极信令网**：所有SP同级直联，适用于规模较小的网络
- **分级信令网**：HSTP（高级信令转接点）→ LSTP（低级信令转接点）→ SP，适用于全国性大网

#### 信令路由

- **正常路由**：正常情况下使用的信令路由
- **替代路由**：正常路由不可用时启用的备用路由
- **选路原则**：优先选择经过STP数量最少的路径（最小跳数原则）

# 第三部分：第二章 宽带IP网络与智能网

## 2.1 宽带IP网络概述

### 网络综合演进历程

PSTN（20世纪70年代）→ ISDN（80年代初）→ ATM（90年代）→ 宽带IP（90年代初至今）

- **ISDN**：通信能力无数量级提高，终端成本高，未大规模普及
- **ATM**：实现过于复杂、应用价格高、桌面业务发展缓慢
- **宽带IP**：Internet因Web技术获得极大成功，"选择IP并非它足够理想，而是因为它已经无处不在"

**宽带IP网定义**：运行实时业务时能保证服务质量（QoS）的IP网；交换机、中继通信线路、用户终端都是宽带化的；中继线带宽几至几十Gbps，接入带宽1-100Mbps。

**实现宽带IP需解决的两个关键问题**：

1. 宽带化（传输 + 交换）
2. 提供可靠的QoS

---

## 2.2 QoS基础

### QoS定义

QoS（服务质量）是服务性能的总效果，决定用户对服务的满意程度。

### 电话通信网QoS指标

- **传输质量**：响度、清晰度、逼真度
- **接续质量**：呼损、接续时延
- **稳定质量**：失效率、平均故障间隔时间（MTBF）、平均故障修复时间（MTTR）

### 数据通信网QoS指标

业务可用性、延迟（Latency）、延迟抖动（Jitter）、吞吐量、丢包率、差错率、响应时间、连接建立时间、故障检测和改正时间

### IP QoS五大技术模块

**标签 → 分类 → 管制 → 调度 → 呼叫接纳**

### 调度机制（排队规则）

| 算法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **FIFO**（先进先出） | 按到达顺序服务 | 简单 | 无法区分优先级，不公平 |
| **严格优先级排队** | 高优先级绝对优先 | 保证高优先级 | 低优先级可能饿死 |
| **WFQ**（加权公平排队） | 单位时间发送量与权重成正比 | 最好时延性能，流隔离 | 时间复杂度高 |
| **WRR**（加权轮转） | 轮转调度，权重决定发送量 | 不需时标比较，实现简单 | 不能提供延时保证 |
| **CBQ**（基于级别的排队） | 分级树状结构 | 灵活分类 | — |

- **IntServ采用的调度算法**：WFQ（严格隔离流并保障带宽时延）
- **DiffServ采用的调度算法**：EF用PQ（绝对优先级队列），AF用CBWFQ

### 管制机制参数

平均速率、峰值速率、突发长度

### 提供QoS的四大方案

1. **综合业务模型（IntServ）**——资源预留
2. **区分服务模型（DiffServ）**——边缘分类 + 核心转发
3. **多协议标签交换（MPLS）**——标签快速转发 + 流量工程
4. 流量工程和约束路由

---

## 2.3 综合业务模型（IntServ）【重点】

### 基本思想

**资源预留**——通信前先为业务流预留沿途网络资源

### 核心概念——流（Flow）

具有同样的源IP、源端口、目的IP、目的端口和协议标识符的一连串分组

### 两个主要特点

1. 资源预留
2. 呼叫建立

### 三类服务

| 服务类别 | 英文 | 特性 | 保障级别 |
|---------|------|------|---------|
| **有保证的服务** | Guaranteed Service（GS） | 提供确定带宽和端到端延时，保证合法数据包无排队丢失 | 绝对QoS保证 |
| **受控负载的服务** | Controlled-Load Service（CLS） | 无固定延时保证，但性能不随网络负载增加而下降（重载时仍如轻载） | 近似QoS |
| **尽力而为的服务** | Best Effort（BE） | 无QoS保证，尽力转发 | 无QoS |

### 四个功能部件【必背！2023简答5、2025简答5】

| 功能部件 | 功能 | 考试重点 |
|---------|------|---------|
| **RSVP**（资源预留协议） | 由接收方控制的带内信令；发送方发PATH报文，接收方回复RESV报文沿途预留资源 | 2023简答5直接考核 |
| **接纳控制**（Admission Control） | 基于服务协议对用户访问进行监视和控制，决定是否接受新流（不降低已有流QoS前提下） | 2025简答5直接考核 |
| **分类器**（Classifier） | 根据规则将进入路由器的分组分类，放入特定队列 | — |
| **调度器**（Scheduler） | 根据QoS要求对分组队列进行发送服务调度（通常用WFQ） | — |

### RSVP工作原理

1. 发送方发送PATH报文（含业务流规格说明TSpec）到目的地
2. 沿途路由器记录路径状态
3. 接收方收到PATH后，沿反向路径发送RESV报文
4. RESV报文沿途在各路由器中预留资源
5. 属于**接收方控制**的带内信令

### IntServ优点

- 能提供绝对有保证的QoS
- 支持单播和多播
- 适用于企业局域网 / 接入网

### IntServ致命缺点

- **状态信息数量与流数目成正比**（可扩展性差）
- 核心网络流数巨大时状态爆炸
- 体系结构复杂
- 不适合短期流
- 自动识别分类不成熟

---

## 2.4 区分业务模型（DiffServ）【重点！2024简答5、2025简答6】

### 核心思想

- 将IPv4 ToS字段 / IPv6 Traffic Class字段重新定义为**DS字段**（区分业务字段）
- DS字段使用前6 bit，即**DSCP**（区分业务码点）
- **复杂性放在边界路由器**，核心路由器工作尽可能简单
- 无需信令协议，核心层不维护每流状态

### 三类业务【必背！】

| 业务类型 | 全称 | DSCP | 特性 | QoS特性 |
|---------|------|------|------|---------|
| **EF**（加速转发 / 优质业务PS） | Expedited Forwarding / Premium Service | 101110（46） | 低延时、低抖动、低丢失率、保证带宽（"虚拟专线"） | 静态QoS |
| **AF**（确保转发 / 确保业务AS） | Assured Forwarding / Assured Service | 4个等级 x 3个优先级 = 12个码点 | 预定最小保证值；超出速率部分按丢弃优先级丢包 | **具有动态QoS特性** |
| **BE**（尽力而为） | Best Effort | 000000（0） | 无QoS保证 | 无QoS |

> **注意**：AF具有**动态QoS特性**——根据丢弃优先级阈值动态丢包（2024简答5直接考！）

### 两种PHB（每跳行为）

**EF PHB**（快速转发）：

- 低丢失率、低时延、低抖动、确保带宽
- 单独一个DSCP码点
- 适用于VoIP、视频会议等实时业务

**AF PHB**（保证转发）：

- DSCP bit0-1划分4个等级（AF1x-AF4x）
- DSCP bit2-5划分3个丢弃优先级（低 / 中 / 高）
- 共12个码点
- 适用于流媒体、商业IP-VPN

### 边界路由器流量调节功能（TC）

分类（Classification）→ **计量**（Metering，令牌桶 / 漏桶算法）→ **标记**（Marking，设置DSCP）→ **整形**（Shaping，平滑突发流量）→ **丢弃**（Dropping，超额丢弃 / 降级转发）

> **业务分类在什么节点进行？**在网络的**边缘节点**进行！（2024考题）

### 边界路由器两类分类器

- **MF分类器**（多域）：基于多个字段（IP地址、端口等）分类，用于边界路由
- **BA分类器**（行为聚合）：仅根据DS值分类，用于核心路由

### 拥塞处理机制

- **RED**（随机早期检测）：随机丢弃分组，避免TCP全局同步
- **WRED**（加权随机早期检测）：对不同优先级采用不同丢弃概率

### IntServ vs DiffServ 综合对比表【必背！2023论述5】

| 维度 | IntServ（综合业务） | DiffServ（区分业务） |
|------|-------------------|---------------------|
| **粒度** | 每流（Per-Flow） | 每类（Per-Class） |
| **可扩展性** | 差（每流状态，状态爆炸） | 好（仅边缘复杂，核心无状态） |
| **信令协议** | RSVP（需要信令） | 无需信令 |
| **QoS保证** | 确定性（绝对QoS） | 统计性（相对QoS） |
| **实现复杂度** | 高（全路由器需支持RSVP + 状态管理） | 中（边缘为主，核心简单转发） |
| **状态维护** | 路由器逐流维护状态 | 核心层不维护流状态 |
| **适用场景** | 企业局域网 / 接入网 | 运营商骨干网 / 核心网 |
| **调度算法** | WFQ | EF用PQ，AF用CBWFQ |

**记忆口诀**：

- IntServ = "每流预留，绝对保证但扩展差"
- DiffServ = "边缘分类打标记，核心简单无状态"

### IntServ与DiffServ互通方案

综合业务覆盖在区分业务网上（RSVP信令透明通过DiffServ域）；并行处理

## 2.5 MPLS（多协议标签交换）【三年连续考查！★★★】

### MPLS基本概念

**MPLS思路**：借鉴ATM / 帧中继标签功能，给IP分组打上标签；将**路由控制和分组转发分离**

**被称为"2.5层技术"**——位于第2层（数据链路层）和第3层（网络层）之间

### 核心术语【必背！】

| 术语 | 全称 | 含义 |
|------|------|------|
| **Label** | 标签 | 短的、定长的、只有本地意义的标识符（32 bit） |
| **LSR** | 标签交换路由器（Label Switch Router） | MPLS核心路由器，基于标签高速转发 |
| **LER** | 边缘标签路由器（Label Edge Router） | 入口压入标签（PUSH），出口弹出标签（POP） |
| **FEC** | 等价转发类（Forwarding Equivalence Class） | 以相同方式转发的一组IP分组；FEC与Label一一对应 |
| **LSP** | 标签交换路径（Label Switched Path） | 从入LER到出LER的单向标签交换路径 |
| **LDP** | 标签分发协议（Label Distribution Protocol） | LSR之间分发标签-FEC绑定关系的控制协议 |

### MPLS标签头结构（32比特）

| 字段 | 位数 | 说明 |
|------|------|------|
| LABEL（标签值） | 20 bit | 标签值，0-15保留 |
| COS / EXP（业务类别） | 3 bit | 用于QoS（对应IP优先级） |
| S（堆栈底标识） | 1 bit | S=1表示这是标签栈底部 |
| TTL（生存时间） | 8 bit | 用于防止循环（同IP TTL） |

标签位于第2层帧头和第3层IP头之间（"垫片层 / Shim Header"）

### 标签分发方式【重点！2023简答6下游、2024简答6上游】

#### 下游标签分配（2023考题）

- 由LSP**末端节点向上游方向**分配标签
- 分为两种：
  - **下游按需分发（DOD）**：上游请求后，下游才分配标签
  - **下游自主分发（DU）**：下游主动向所有上游分发标签绑定
- 优点：不会产生标签重复问题

#### 上游标签分配（2024考题）

- 由LSP**起始节点向下游方向**分配标签
- 上游自主决定标签后通知下游
- 适用于需要特定QoS保障的场景

#### 标签分发控制方式

| 方式 | 说明 |
|------|------|
| **独立控制** | 各LSR独立决定何时向上游分发标签（快但可能不一致） |
| **有序控制** | 下游LSR先分发，上游后分发，形成级联（慢但一致） |

### MPLS工作流程（2025论述5 MPLS组网原理）

```
[IP网络] → [入口LER: PUSH标签] → [LSR: SWAP标签] → [LSR: SWAP标签] → [出口LER: POP标签] → [IP网络]
|←———————————————————— LSP（标签交换路径） ——————————————————————————→|
```

**四步工作流程**：

1. LDP和传统路由协议在各LSR之间建立标签映射表和转发表（建立LSP）
2. 入口LER接收IP包，完成第3层功能（FEC分类），为分组**加上标签（PUSH）**
3. 核心LSR仅依据标签值，通过硬件标签转发表**快速交换转发（SWAP）**
4. 出口LER**去除标签（POP）**，恢复为标准IP包继续转发

### PHP（次末跳弹出）

出口LER的前一跳LSR就弹出标签，避免出口LER做双重查表（标签查表 + IP查表）

### MPLS解决的问题（作业总结）

- 传统IP逐跳查表转发**速度慢**
- 缺乏**流量工程**能力（最短路径导致部分链路拥塞、部分空闲）
- QoS**难以保证**

**解决方案**：标签交换快速转发 + 面向连接LSP显式路由控制 + 2.5层多协议兼容

### MPLS优点与缺点

**优点**：

- 转发简单快速（仅查标签表）
- FEC增强可扩展性
- 支持基于QoS的路由
- 流量管理（流量工程TE）
- 分布式处理
- 支持VPN（双层标签栈：外层隧道 + 内层VPN标签）

**缺点**：

- 短流信令开销大
- 标签交换机仍需传统IP分组转发功能
- FEC自动识别技术有待提高

### MPLS典型应用场景

- 流量工程（TE）
- MPLS VPN（L2 VPN / L3 VPN）
- QoS保障
- 快速重路由（FRR）保护

## 2.6 路由算法【三年均考！★★★】

### 路由算法两大核心功能

1. 为不同源节点和目的节点对（SD）**选择一条传输路径**
2. 选定路径后，**将用户消息正确传送到目的节点**

第二项功能使用**路由表**实现。路由表记录从当前节点到达目的节点的下一跳（或输出链路），以及路由质量 / 利用率的度量值。

### 路由选择策略（习题课重点）

| 传输方式 | 路由选择策略 | 说明 |
|---------|------------|------|
| **数据报**（Datagram） | 对每个分组**都重新**作路由选择 | 灵活，但开销大 |
| **虚电路**（Virtual Circuit） | 仅在建立虚电路时作**一次**路由选择，之后沿已建立路由传送 | 效率高，但灵活性差 |

### 路由算法分类（五大维度）

| 分类维度 | 类别 |
|---------|------|
| 决策地点 | 每一节点、中央节点、源节点、节点子集 |
| 决策时间 | 分组（数据报）、会话（虚电路） |
| 性能准则 | 链路数、设施代价、时延、吞吐量 |
| 网络信息源 | 无、本地、相邻节点、路径上的节点、所有节点 |
| 路由选择策略 | **静态**（非自适应，简单开销小）/ **自适应**（连续变更 / 周期性变更 / 主要负载改变时 / 拓扑改变时） |

### 集中式 vs 分布式路由（2024简答4）

| 类型 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **集中式路由** | 由路由控制中心（RCC）计算，周期收集各链路状态，周期下发路由表给各节点 | 全局最优 | 单点故障风险 |
| **分布式路由** | 所有节点通过相互交换路由信息，**独立地**计算到达各节点的路由 | 鲁棒，适合大规模 | 近优非全局最优 |

### 广域网路由 vs 互联网路由（2023简答3）

- **广域网路由**：解决一个**子网内**的路由；常用算法：Dijkstra、距离矢量
- **互联网路由**：解决**不同子网之间**的路由；常用协议：BGP

### 理想路由算法五大特点【必背！】

| 特点 | 说明 |
|------|------|
| **正确性** | 分组一定能到达目的节点；到达后不会再向其他节点转发 |
| **计算简单** | 使用最少运行资源，节省开销减少时延，尽量少使用节点间链路带宽 |
| **自适应性（鲁棒性）** | 适应网络业务量和拓扑变化；节点 / 链路故障时能及时找到替换路径 |
| **稳定性** | 算法必须**收敛**；不会在多条可能路径间来回不停变化（无过度振荡） |
| **公平性** | 对所有用户等同；不能仅使某一对用户端到端时延最小而占用过多资源 |

**记忆口诀**：**正简自稳公**——正确性、计算简单、自适应性、稳定性、公平性

> 实际约束：**没有算法能全部满足**，有的要求可能矛盾（如吞吐量最大可能增加时延）。

### Dijkstra算法（链路状态路由）【2025简答3！】

**基本原理**：通过逐步扩展已确定最短路径的节点集合，最终获得从源节点到所有其他节点的最短路径。

**算法步骤**：

1. 初始化：N' = {源节点u}，对u的每个邻居v：D(v) = c(u,v)；对其他节点w：D(w) = ∞
2. 循环直至N'包含所有节点：
   - (a) 从不在N'中的节点中选取D(w)最小的节点w，加入N'
   - (b) 对w的每个不在N'中的邻居v：更新 D(v) = min(D(v), D(w) + c(w,v))
3. 复杂度：O(n²)（朴素实现）或 O(n log n + m)（堆优化）

**手算示例**（以以下网络为例，源节点为A）：

```
网络拓扑：
    B —— 3 —— C
   /|         |\
  2 |         5 | 2
 /  |         |  \
A — 1 — D — 2 — F
         |       |
         4       1
         |       |
         E — 3 — G

链路代价：A-B=2, A-D=1, B-C=3, B-D=1, C-F=5, D-E=4, D-F=2, E-G=3, F-C=2, F-G=1
```

**Dijkstra算法逐步执行**：

| 步骤 | N'（已确定最短路径的节点集） | D(B),前驱 | D(C),前驱 | D(D),前驱 | D(E),前驱 | D(F),前驱 | D(G),前驱 |
|------|--------------------------|-----------|-----------|-----------|-----------|-----------|-----------|
| 0 | {A} | 2, A | ∞, — | 1, A | ∞, — | ∞, — | ∞, — |
| 1 | {A, D} | 2, A | ∞, — | — | 5, D | 3, D | ∞, — |
| 2 | {A, D, B} | — | 5, B | — | 5, D | 3, D | ∞, — |
| 3 | {A, D, B, F} | — | 5, B/F | — | 5, D | — | 4, F |
| 4 | {A, D, B, F, G} | — | 5, B/F | — | 5, D | — | — |
| 5 | {A, D, B, F, G, C} | — | — | — | 5, D | — | — |
| 6 | {A, D, B, F, G, C, E} | — | — | — | — | — | — |

**A的路由表（转发表）**：

| 目的节点 | 下一跳 | 最短距离 |
|---------|--------|---------|
| B | B | 2 |
| C | D | 5 |
| D | D | 1 |
| E | D | 5 |
| F | D | 3 |
| G | D | 4 |

### 距离矢量路由算法【2025论述2！】

**基本原理**：每个节点仅与**邻居**交换距离向量（DV），根据邻居信息迭代更新自己的路由表。

**Bellman-Ford方程**：

$$D_x(y) = \min_v \{ c(x,v) + D_v(y) \}$$

其中v遍历x的所有邻居，D_x(y)是节点x到节点y的最小代价估计。

**算法步骤**：

1. 初始化：节点x获知到所有直接邻居的代价；对其他节点估计为∞
2. 每个节点周期性地向所有邻居发送自己的距离向量（或向量变化时立即发送）
3. 收到邻居v的距离向量后：对每个目的y，计算 c(x,v) + D_v(y)，若更小则更新自己的D_x(y)
4. 重复直至收敛

**计数到无穷问题**【重点！】：

- 当链路代价突然增大（如链路故障），"坏消息传得慢"
- 例如：x-y链路断开（代价→∞），x和y各自认为通过对方可以到达，互相通告，距离逐轮+1，需很多轮才能收敛
- **解决方案**：**毒性逆转（Poisoned Reverse）**
  - 如果z通过y路由到达x，则z在给y的DV中通告D_z(x) = ∞（"下毒"）
  - 防止y在z→x的路由上选择z作为下一跳

### LS（链路状态）vs DV（距离矢量）对比表【必背！】

| 维度 | 链路状态（LS / Dijkstra） | 距离矢量（DV / Bellman-Ford） |
|------|--------------------------|-------------------------------|
| **代表协议** | OSPF, IS-IS | RIP, BGP |
| **信息交换范围** | 向全网洪泛链路状态 | 仅与邻居交换DV |
| **收敛速度** | 快 | 慢（计数到无穷问题） |
| **网络开销** | 较高（n条LSP洪泛） | 较低（仅与邻居交换） |
| **扩展性** | 好（分层OSPF：骨干区 + 普通区） | 差（RIP有15跳限制） |
| **可靠性** | 高（各节点独立计算） | 低（错误消息会传播） |
| **路由计算** | 全局计算（Dijkstra） | 分布式迭代（Bellman-Ford） |

### 路由选择对网络性能的影响（2024论述1）

- 根据不同的路由选择方法，网络可接纳的最大通过量可达**10~30个单位**（相差3倍）
- 在高业务负荷下：保证相同时延条件下**增加网络通过量**
- 在轻负荷和中等负荷下：**减少每个分组的平均时延**
- 路由表需根据网络运行情况加以修改、更新

**多源节点路由分析例题**（习题课重点）：

- 场景：2个源节点（1和2）、1个目的节点（6）、链路容量均 = 10单位
- λ1 = λ2 = 5时：好方案（分散路径，链路50%利用率 → 时延小）；差方案（都经同一链路 → 满载 → 时延大）
- λ1 = 5, λ2 = 15时：好方案（节点2分摊到两条路径）；差方案（节点2只用一条路径 → 需丢弃5单位）

## 2.7 智能网（IN）【2023简答7、2024简答7、2025简答7！★★★】

### 智能网基本概念

**定义**：在原有通信网络基础上为用户提供新业务而设置的**附加网络结构**，用于产生和提供新业务的体系概念。

**核心思想**：**业务逻辑与基础通信功能分离**——交换层仅处理基本呼叫连接，复杂的增值业务控制交给智能层。

**智能网层次结构（三层）**：

1. **传输层**：交换设备、传输设备、SSP（业务交换点）、中继线路、IP（智能外设）
2. **信令层**（连接层）：公共信道信令网络（No.7信令网、STP）
3. **智能层**：SCP（业务控制点）、SMS（业务管理系统）、SCE（业务生成环境）

### 智能网概念模型（INCM）——四个平面【必背！】

```
┌── 业务平面（SP） ————————————————————————————————┐
│  面向用户，定义智能网业务和业务特征                  │
│  IN CS-1：25种业务、38种业务特征                    │
│  如：免费电话 = "单一号码" + "反向计费"两个业务特征   │
├── 全局功能平面（GFP） ——————————————————————————————┤
│  面向业务设计者，将智能网视为一个整体                 │
│  核心：SIB（业务独立构件）+ GSL（全局业务逻辑）      │
│  BCP（基本呼叫处理）：SIB链的起点                    │
├── 分布功能平面（DFP） ——————————————————————————————┤
│  面向网络设计者，描述功能实体的分布与交互              │
│  9个功能实体FE + 功能实体动作FEA + 信息流IF           │
├── 物理平面（PP） ——————————————————————————————————┤
│  面向网络实施者，将DFP中的FE映射到实际物理设备        │
│  6种物理实体 + INAP协议接口                          │
└──────────────────────────────────────────────────┘
```

### 四个平面详细说明

| 平面 | 面向对象 | 核心内容 | 关键概念 |
|------|---------|---------|---------|
| **SP**（业务平面） | 用户 | 业务与业务特征的定义 | 25种业务、38种业务特征 |
| **GFP**（全局功能平面） | 业务设计者 | 业务逻辑的全局描述 | SIB（业务独立构件）、GSL（全局业务逻辑）、BCP（基本呼叫处理） |
| **DFP**（分布功能平面） | 网络设计者 | 功能实体的分布式架构 | 9个FE：CCF / SSF / SCF / SDF / SRF / SCEF / SMF / SMAF / CCAF |
| **PP**（物理平面） | 网络实施者 | 功能到物理的映射 | SCP / SSP / SCE / SMS / STP / IP等物理实体 |

### SIB（业务独立构件）

- 标准化的、可重用的功能构件
- 涵盖：鉴权、计算、号码翻译、用户交互、链接、数据查询、计费等
- **GSL（全局业务逻辑）**：将SIB按一定顺序组合链接形成智能网业务

|------|---------|------|

| STP（信令转接点） | — | No.7信令网一部分；成对配置以提高可靠性 |
| IP（智能外设） | SRF | 提供特殊资源如：语音合成、DTMF接收、语音识别、录音通知等 |

### INAP（智能网应用协议）【作业总结重点！】

- **定义**：SSP与SCP之间的**高层标准信令协议**，实现业务逻辑与交换功能的解耦
- 位于No.7信令的应用层（用户部分）
- 采用抽象文法记法**ASN.1**描述
- 调用**TCAP**协议进行实际传输
- 由ITU-T Q.12x8系列建议规范

**INAP操作类型**：

1. 既报告成功也报告失败（如InitialDP）
2. 只报告成功（如PromptAndCollectUserInformation）
3. 只报告失败
4. 既不报告成功也不报告失败

**核心操作**：

| 操作 | 方向 | 说明 |
|------|------|------|
| InitialDP（初始检测点） | SSP → SCP | 报告智能呼叫被检测到 |
| Connect（连接） | SCP → SSP | 命令建立连接（含翻译后的真实号码） |
| PromptAndCollectUserInformation（提示并收号） | SCP → SSP | 要求向用户播放提示音并收集DTMF输入 |
| ReleaseCall（释放呼叫） | SCP → SSP | 命令释放呼叫 |

### 800号业务处理流程【必掌握】

```
1. 用户拨800-XXXXXXX → 端局交换机 → SSP
2. SSP识别800前缀 → 触发智能业务 → 发送InitialDP给SCP（含主叫号码、被叫800号码）
3. SCP查询数据库 → 将800号码翻译为真实物理号码 → 返回Connect消息给SSP
4. SSP根据真实号码 → 完成后续电路连接 → 通话建立
```

### 智能网业务举例

被叫集中付费（800号）、记账卡业务（300号）、虚拟专用网（VPN）、预付费业务（PPS）、通用个人通信（UPT）、大众呼叫、缩位拨号、呼叫转移

> 考试写出4-5个即可

### 智能网 vs 非智能网处理方式（作业总结）

| 对比维度 | 非智能网 | 智能网 |
|---------|---------|--------|
| **业务逻辑位置** | 固化在交换机软件中 | 集中在SCP中 |
| **增加新业务** | 需修改所有交换机代码，周期长、成本高 | 仅在SCP部署新业务逻辑（SLP），周期短 |
| **业务与交换关系** | 紧密耦合 | **分离** |
| **网络开放性** | 封闭 | 开放（支持第三方开发） |
| **灵活性** | 低 | 高 |

## 2.8 IP网络安全（作业总结补充）

### IP网络面临的安全问题

1. **IP欺骗**（篡改源IP绕过认证）
2. **DDoS攻击**（分布式拒绝服务，耗尽带宽 / 资源）
3. **数据窃听与篡改**（IP明文传输）
4. **路由协议攻击**（伪造路由信息）
5. **网络扫描与探测**

### 安全防护机制

| 防护层面 | 机制 | 说明 |
|---------|------|------|
| **网络层** | 防火墙、ACL | 包过滤，控制进出流量 |
| **传输层** | TLS / SSL | 端到端加密传输 |
| **网络层** | IPsec | VPN隧道加密，AH认证头 + ESP封装安全载荷 |
| **应用层** | 入侵检测 / 防御系统（IDS / IPS） | 深度包检测，识别攻击特征 |
| **管理层面** | AAA（认证、授权、计费） | RADIUS / Diameter协议 |

### 防火墙类型

| 类型 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **包过滤防火墙** | 根据IP头、TCP/UDP头信息过滤 | 简单、速度快 | 无法检测应用层攻击 |
| **状态检测防火墙** | 维护连接状态表，动态放行回程流量 | 安全性增强 | 性能开销较大 |
| **应用代理防火墙** | 代理应用层请求，深度检查内容 | 安全性最高 | 速度慢，不支持所有应用 |

### IPsec两种工作模式

| 模式 | 加密范围 | 适用场景 |
|------|---------|---------|
| **传输模式** | 仅加密IP数据载荷 | 端到端通信 |
| **隧道模式** | 加密整个原始IP包（含IP头），外加新IP头 | VPN网关间通信 |

# 第四部分：第三章 下一代网络与软交换

## 3.1 NGN概述

### NGN定义
NGN（Next Generation Network）泛指一个不同于目前一代的、以IP为中心的**全业务网络**，可以支持语音、数据和多媒体业务的融合，支持固定接入和移动接入。

关键理解：NGN的出现与发展**不是革命，而是演进**。

### 不同网络视角的NGN
| 现有网络 | NGN方向 |
|---------|---------|
| 传输网络 | SDH/DWDM → ASON（自动交换光网络） |
| 计算机网络 | IPv4 Internet → IPv6 NGI（下一代Internet） |
| 移动网络 | GSM → 3G → 4G → 5G → 6G |
| 电话网 | TDM程控交换 → 软交换 |
| 卫星通信网 | 与地面网络融合 → 6G空天地一体化 |

### NGN希望具备的特性
传统电话网的普遍性和可靠性 + 因特网的灵活性 + 以太网的运作简单性 + ATM的低延时 + 光网络的带宽 + 蜂窝网的移动性 + 有线电视网的丰富内容

### 为什么需要发展NGN
**现有网络的问题**：
- 多业务网（PSTN/Internet/FR/ATM）并存，独立发展
- 功能单一、运维分立
- 负荷不均衡、资源无法共享
- 互联互通困难
- 重复投资、设备集成度低

**运营趋势**：
- 降低总体运维成本(CAPEX+OPEX)
- 业务丰富、实现快速
- 整合网络资源
- 可持续演进

### NGN四大核心特征
1. **统一的分组交换网络平台**（IP核心）
2. **分布式的网络体系结构**
3. 同时支持语音、数据、视频等多种业务
4. **业务与控制分离、控制与连接（承载）分离**

长远综合建网成本远低于PSTN。

### NGN功能分层结构（四层架构）—— 必画图！

```
┌─────────────────────────────────────────────────┐
│  第4层：业务层（应用层）                         │
│  应用服务器(AS), 媒体服务器(MS), AAA服务器, SCP   │
│  协议：SIP, Parlay, INAP                        │
│  功能：提供各种增值业务、业务和网络管理            │
├─────────────────────────────────────────────────┤
│  第3层：控制层                                   │
│  软交换设备(Softswitch/MGC)                      │

### NGN网络特点与优势
- 业务与呼叫控制分离，呼叫与承载分离
- 业务升级与基础设备无关（硬件软件化）
- 提供可扩展的业务平台，业务投放周期短
- 提供开放的业务接口(API)给第三方
- 地域概念消失（IP的全球可达性）

## 3.2 软交换（Softswitch）

### 软交换概念起源
起源于美国企业网环境下的IP PBX——基于以太网电话，通过PC服务器的呼叫控制软件实现PBX功能。

### 软交换核心思想
将传统交换设备**部件化**，分为呼叫控制与媒体处理两部分，二者间采用标准协议，且主要使用**纯软件**处理。

### 软交换基本含义（2023简答8）
将**呼叫控制功能从媒体网关中分离出来**，通过软件实现基本呼叫控制功能，实现**呼叫传输与呼叫控制的分离**。

### 软交换三大特点
1. 通过**软件方式**实现交换机的控制、接续和业务处理功能
2. **呼叫控制与承载（媒体）分离**
3. **业务控制与呼叫控制分离**

两个分离 = NGN的核心理念！

### 软交换六大功能

| 功能 | 说明 |
|------|------|
| **媒体接入功能** | 通过H.248协议接入各类媒体网关；通过H.323和SIP协议接入各类终端 |
| **呼叫控制功能** | 确定用户权限、控制网关资源、路由选择、连接状态监视 |
| **业务提供功能** | 实现PSTN/ISDN全部业务；与智能网配合提供增值业务；提供开放API |
| **互连互通功能** | 支持H.323和SIP互通；与智能网及PSTN/ISDN互通 |
| **资源管理功能** | 对带宽等网络资源进行分配和管理 |
| **认证和计费** | 对接入设备进行认证授权、地址解析；支持按流量/时长等计费方式 |

### 软交换在NGN中的作用
- **呼叫控制核心**：整个NGN的"大脑"
- **协议适配枢纽**：完成H.248/SIP/H.323/BICC等多协议互通
- **业务交换平台**：连接智能网SCP和应用服务器AS
- **网络互通桥梁**：连接PSTN、PLMN、IP网等异构网络

### 软交换优点与不足
**优点**：呼叫控制与分组承载分离；方便新业务产生；新老运营公司均有利

**不足**：协议众多且相互竞争；不同厂家互操作问题；QoS保障仍不完善；业务生成和收入能力有待提高

## 3.4 NGN协议体系（重点！）

### 六类协议体系

| 类别 | 协议 | 应用位置 |
|------|------|---------|
| 呼叫控制协议 | H.323, SIP, SIP-T, BICC | 建立/修改/释放会话 |
| 媒体控制协议 | H.248/Megaco, MGCP | 软交换控制媒体网关 |
| 传输控制协议 | RTP, RTCP | 媒体流传输 |
| 信令传输协议 | M3UA, M2UA, SCTP (SIGTRAN协议族) | No.7信令 over IP |
| 业务应用协议 | PARLAY, JAIN, INAP | 业务层开放接口 |
| 维护管理协议 | SNMP, CORBA, COPS | 网络管理与维护 |

### 核心协议详解

#### H.248/Megaco（媒体网关控制协议）
- **应用位置**：软交换(MGC) ↔ 媒体网关(MG)
- **定位**：网关控制协议（主从关系，MGC为主、MG为从）
- **特点**：协议简单、功能强大、扩展性好
- **基本命令**：ADD（添加端点）、MODIFY（修改）、SUBTRACT（删除）、MOVE（移动）、NOTIFY（MG通知MGC）、AUDIT（审计）

#### SIGTRAN（信令传输协议）
- **应用位置**：软交换 ↔ 信令网关(SG)
- **功能**：将PSTN No.7信令转换为IP网对应信令协议
- **采用SCTP**（流控制传输协议）：综合了TCP的可靠性和UDP的实时性优点
- 包含：M3UA(MTP3用户适配)、M2UA(MTP2用户适配)等

#### H.323（多媒体通信系统协议）
- **来源**：ITU-T标准，成熟稳定
- **定位**：传统IP电话/VoIP多媒体通信协议
- **四种构件**：
  1. H.323终端（用户端设备）
  2. 网关(Gateway)：连接H.323网络与非H.323网络
  3. **网守(Gatekeeper)**：地址转换、授权认证、带宽管理、计费
  4. 多点控制单元(MCU)：多方会议控制
- 特点：静态组网，适合大规模部署
- 缺点：复杂度高，逐渐被SIP取代

#### SIP（会话发起协议）——详见3.5节

#### BICC（与承载无关的呼叫控制）
- **应用位置**：软交换 ↔ 软交换（不同软交换域间互通）
- 源自ISUP，实现承载与控制分离的软交换机间互通

### H.248 vs SIP 对比表（2024直接考题！）

| 维度 | H.248/Megaco | SIP |
|------|-------------|-----|
| **定位** | 网关控制协议（主从关系） | 对等会话控制协议 |
| **控制模型** | 垂直层次化控制（MGC→MG下发指令） | 端到端对等交互 |
| **功能** | MGC对MG下发控制指令(ADD/SUBTRACT等) | 建立/修改/释放多媒体会话 |
| **消息编码** | ASN.1二进制 或 文本编码 | 文本编码（类HTTP语法） |
| **标准组织** | ITU-T + IETF联合 | IETF |
| **适用场景** | NGN核心控制层（MGC↔MG） | NGN业务层+终端+软交换间 |
| **协议关系** | 相当于MG的"遥控器" | 相当于会话的"协商者" |

## 3.5 SIP协议与呼叫流程（必考画图！）

### SIP概述
SIP（Session Initiation Protocol，会话发起协议）是IETF制定的应用层信令控制协议，用于创建、修改和释放多媒体会话。

### SIP特点
- 基于HTTP风格：文本编码、URI寻址（如sip:user@domain.com）
- 客户端/服务器模型
- 独立于底层传输协议（一般用UDP减少时延，也可用TCP）
- 使用SDP（会话描述协议）作为消息体描述媒体信息
- 简单灵活、分布式、动态组网

### SIP核心元素

| 元素 | 说明 |
|------|------|
| **UA**（用户代理） | User Agent，SIP终端（UAC发起请求、UAS响应请求） |
| **Proxy Server**（代理服务器） | 路由SIP消息，可进行策略控制 |
| **Registrar Server**（注册服务器） | 接收REGISTER请求，记录用户当前位置信息 |
| **Redirect Server**（重定向服务器） | 向主叫返回被叫的新位置（302），主叫重新发起请求 |

### SIP请求方法（6种核心METHOD）

| 方法 | 功能 | 说明 |
|------|------|------|
| **INVITE** | 发起会话邀请 | 携带SDP描述主叫媒体能力，是最核心的请求 |
| **ACK** | 确认最终响应 | 与INVITE配套使用，仅确认INVITE的最终响应（三方握手） |
| **BYE** | 释放已建立会话 | 任一方挂机时发送 |
| **CANCEL** | 取消未完成请求 | 在收到最终响应之前取消（如被叫未应答前主叫挂机） |
| **REGISTER** | 向Registrar注册 | UA开机/移动时注册当前位置信息 |
| **OPTIONS** | 查询服务器能力 | 探测对端支持的SIP方法和能力，不建立会话 |

### SIP响应类别

| 类别 | 范围 | 含义 | 典型消息 |
|------|------|------|---------|
| 1xx | Provisional（临时） | 请求已收到，正在处理 | 100 Trying（正在处理）, 180 Ringing（被叫振铃） |
| 2xx | Success（成功） | 请求成功处理 | 200 OK（应答/注册成功/释放确认） |
| 3xx | Redirection（重定向） | 需进一步动作 | 302 Moved Temporarily（临时迁移） |
| 4xx | Client Error（客户端错误） | 请求有误或无法处理 | 404 Not Found（用户不存在）, 486 Busy Here（用户忙） |
| 5xx | Server Error（服务器错误） | 服务器内部错误 | 500 Server Internal Error |
| 6xx | Global Failure（全局错误） | 任何服务器都无法处理 | 603 Decline（拒绝） |

### SIP基本呼叫流程时序图（必须能徒手画出！）

主叫UA_A呼叫被叫UA_B，通过Proxy Server：

```
UA_A（主叫）          Proxy Server          UA_B（被叫）
    |                    |                    |
    |——INVITE(SDP_A)——→|                    |   (1) 主叫发起会话邀请
    |                    |                    |      携带主叫支持的媒体编码列表
    |←——100 Trying—————|                    |   (2) Proxy确认收到请求
    |                    |——INVITE(SDP_A)——→|   (3) Proxy转发邀请给被叫
    |                    |                    |
    |                    |←——180 Ringing————|   (4) 被叫开始振铃
    |←——180 Ringing————|                    |   (5) 回转铃音告知主叫
    |                    |                    |
    |                    |←————200 OK(SDP_B)—|   (6) 被叫摘机应答
    |                    |                    |      携带被叫选定的媒体编码
    |←————200 OK(SDP_B)—|                    |   (7) 回传被叫媒体能力
    |                    |                    |
    |=================== ACK ===================→|   (8) 主叫确认（三方握手完成）
    |                    |                    |
    |←—————— 双向RTP媒体流（语音/视频通话）——————→|
    |                    |                    |
    |                    |                    |   (9) 任一方挂机
    |——BYE—————————————→|——BYE—————————————→|
    |                    |                    |
    |                    |←————200 OK————————|   (10) 确认释放
    |←————200 OK————————|                    |
    |                    |                    |
```

**SDP（会话描述协议）**承载在INVITE和200 OK的消息体中，描述：
- 媒体类型（音频audio/视频video）
- 编码格式（G.711/G.729/H.264）
- IP地址和端口号
- 其他会话参数

### SIP呼叫流程关键步骤说明
1. **INVITE(SDP_A)**：主叫发送邀请，SDP列出自己支持的所有编码（如G.711, G.729, H.264）
2. **200 OK(SDP_B)**：被叫应答，SDP从主叫提供的列表中选择一种编码（协商确定最终编码）
3. **ACK**：主叫收到200 OK后发送ACK确认，三方握手完成，会话建立
4. **RTP媒体流**：ACK之后，双方直接用被叫选定的编码通过RTP传输媒体
5. **BYE + 200 OK**：释放时也需BYE-200 OK确认

### SIP与H.323对比
| 维度 | SIP | H.323 |
|------|-----|-------|
| 标准组织 | IETF | ITU-T |
| 编码格式 | 文本（类HTTP） | 二进制（ASN.1） |
| 复杂性 | 简单 | 复杂 |
| 组网方式 | 动态、分布式 | 静态、集中式（需要网守） |
| 扩展性 | 好（HTTP式扩展） | 一般 |
| 终端要求 | 相对智能 | 可较简单 |
| 当前趋势 | 主流，广泛采用 | 逐步被SIP替代 |

# 第五部分：第四章 卫星通信网

## 4.1 卫星通信基本概念

**定义**：利用人造地球卫星作为中继转发或反射无线电波，在多个通信站之间进行通信。

**技术基础**：在地面微波中继通信和空间电子技术基础上发展起来。

---

## 4.2 空天地一体化网络（6G核心方向）

被ITU列为七大关键网络需求之一，6G核心方向。

**四大组成部分**：

- **天基**：卫星通信网络（GEO/MEO/LEO星座）
- **空基**：飞机、热气球、无人机等高空通信平台
- **陆基**：地面蜂窝网络（5G/6G基站）
- **海基**：海洋水下无线通信及近海沿岸无线网络

**6G关键指标**：大部分性能指标相比5G提升10-100倍；峰值速率1Tbps；空口延迟0.1ms；三大应用场景（全覆盖多样化智能连接、高保真扩展现实、智能化行业应用）

## 4.3 通信卫星分类

### 按轨道分类（重点）

| 轨道类型 | 高度 | 运行周期 | 单星覆盖范围 | 延迟 | 代表系统 |
|---------|------|---------|------------|------|---------|
| **GEO**（地球静止轨道） | 35,786km | 24小时 | 约1/3地球表面 | 单程约270ms | 中星系列、Inmarsat、北斗GEO |
| **MEO**（中轨道） | 2,000-20,000km | 2-12小时 | 较大区域 | 中等 | GPS(20,200km)、北斗MEO |
| **LEO**（低轨道） | 200-2,000km | 90min-2h | 较小（几百km） | 低（约25ms） | 星链、OneWeb、鸿雁、虹云、千帆 |

其他轨道类型：赤道轨道、极轨道(SSO太阳同步轨道)、倾斜轨道、GTO（地球同步转移轨道-椭圆轨道）

### 按频段分类（必背，高频考点）

| 频段 | 频率范围 | 主要业务 | 特点 |
|------|---------|---------|------|
| VHF | 0.1-0.3 GHz | 移动、导航业务 | — |
| UHF | 0.3-1.0 GHz | 移动、导航业务 | — |
| **L** | 1.0-2.0 GHz | 移动卫星通信、GPS导航 | 穿透力最强、天线大 |
| **S** | 2.0-4.0 GHz | 移动通信、深空通信、指令传输 | 较好穿透 |
| **C** | 4.0-8.0 GHz（上行6G/下行4G） | 固定卫星业务(FSS) | 雨衰较小、技术成熟、频率拥挤 |
| **X** | 8.0-12.0 GHz | 军用、政府通信 | 预留频段 |
| **Ku** | 12.0-18.0 GHz（上行14G/下行12G） | DBS直播卫星、VSAT | 天线小、雨衰显著 |
| K | 18.0-24.0 GHz | — | — |
| **Ka** | 24.0-40.0 GHz（上行30G/下行20G） | 高通量宽带卫星互联网 | 大带宽、雨衰严重 |
| Q/V | 33.0-75.0 GHz | 未来高通量 | — |

**记忆规律**：L→S→C→X→Ku→Ka，频率从低到高，雨衰越来越大，天线越来越小，带宽越来越大

### 按重量分类

| 类别 | 重量 | 成本（估算） |
|------|------|------------|
| 大型卫星(Large) | >1000 kg | >1亿美元 |
| 小卫星(Small) | 500-1000 kg | 0.5-1亿美元 |
| 微小卫星(Mini) | 100-500 kg | 500-2000万美元 |
| 微卫星(Micro) | 10-100 kg | 200-300万美元 |
| 纳卫星(Nano) | <10 kg | <100万美元 |
| 皮卫星(Pico) | <1 kg | — |

### 按转发能力分类

- **无星上处理能力**：透明转发（弯管式）
- **有星上处理能力**：再生处理（含解调、解码、交换、再调制）

### 按通信范围分类

国际通信卫星、区域性通信卫星、国内通信卫星

### 按用途分类

综合业务通信卫星、军事通信卫星、海事通信卫星、电视直播卫星、气象卫星等

## 4.4 卫星通信特点

### 七大优点

| 优点 | 说明 |
|------|------|
| 通信距离远 | 费用与距离无关，静止卫星最大通信距离18100km |
| 广播方式工作 | 天然广播特性，一点对多点，可进行多址通信 |
| 通信容量大 | C和Ku频段带宽500-800MHz，Ka频段达几个GHz |
| 自发自收监测 | 可接收自己发出的信号进行质量监测 |
| 无缝覆盖能力 | 特别是海洋、沙漠、高山等地面网难以覆盖区域 |
| 广域组网能力 | 灵活多点波束+星上交换处理，构成复杂网络拓扑 |
| 安全可靠性高 | 抗震救灾、光缆故障时的应急备份，独立于地面基础设施 |

### 三大缺点

| 缺点 | 说明 |
|------|------|
| **通信时延较长** | GEO单跳传输时延约**270ms**（影响实时交互） |
| **链路易受外部影响** | 大气层干扰、雨衰（尤其Ku/Ka）、电离层闪烁等 |
| **星蚀和日凌中断** | 星蚀（卫星进入地球阴影，太阳能供电中断）；日凌（太阳直射地面站天线，噪声淹没信号） |

## 4.6 通信卫星的组成

### 两大部分

1. **空间平台**（卫星公用舱/服务舱）：
   - 结构分系统、温控分系统、电源分系统（太阳能电池板+蓄电池）
   - 控制分系统（姿态控制、轨道控制）
   - 遥测分系统、指令分系统
   - 静止轨道卫星还包括远地点发动机

2. **有效载荷**（通信任务载荷）：
   - **天线**：全向天线（遥测遥控信令）、微波定向天线（全球波束/点波束/赋形波束）
   - **转发器**：卫星通信的**核心部件**

### 转发器类型对比（重点）

| 类型 | 工作方式 | 变频方式 | 特点 | 优点 | 缺点 |
|------|---------|---------|------|------|------|
| **透明转发器**（弯管式/Bent-Pipe） | 只放大→变频→再放大，不处理信号内容 | 一次变频（射频→射频，大容量，带宽500MHz）/ 二次变频（射频→中频→射频，增益80-100dB） | 与信号波形和调制方式无关 | 结构简单、技术成熟、低延迟 | 上下行噪声累积叠加、无再生能力 |
| **处理转发器**（再生式） | 解调→数据处理→再调制→上变频→放大 | — | 信息处理型（消除噪声积累）/ 空间路由型（星上交换） | 噪声不累加、抗干扰强、可星上交换路由 | 复杂、昂贵、与波形绑定 |

**混合转发器**：结合透明和再生两者功能。

## 4.7 信号传输基本流程

```
地球站A发送：
  终端设备 → 调制器 → 上变频器 → 功率放大器 → 天线发射（上行频率f1）
                    ↓
卫星转发器处理：
  天线接收 → 低噪声放大器 → 下变频器(f1→中频) → 中频放大器 → 上变频器(中频→f2) → 功率放大器 → 天线发射（下行频率f2）
                    ↓
地球站B接收：
  天线接收 → 低噪声放大器 → 下变频器(f2→中频) → 中频放大器 → 解调器 → 终端设备 → 分路输出给用户
```

# 第六部分：第五章 网络管理与规划

## 5.1 网络管理基本概念

### 网络管理定义

包括对**硬件、软件和人力**的使用、综合与协调，以便对网络资源进行**监视、测试、配置、分析、评价和控制**，能以合理的价格满足用户对网络的需求。

### 管理网定义

一个用于管理和控制通信网络的体系结构，提供标准化的接口和协议，支持各种网络设备的管理和操作。主要功能包括：网络资源管理、性能管理、故障管理、配置管理以及安全管理等。

### 网络管理三大环节

1. 网络管理信息的**采集**
2. 网络管理信息的**传输**
3. 网络管理信息的**处理与决策**

### 网络管理系统组成

- **管理系统(Manager)**：发出管理操作指令、接收告警/事件
- **被管系统**：由代理(Agent)和被管对象(MO)组成
- **基本模型**：Manager — Agent 模型（Manager发出请求，Agent响应请求或主动发送Trap）

### 两大网络管理协议

| 协议 | 来源 | 基础网络 |

## 5.5 TMN（电信管理网）——三年均考

### TMN为什么被提出

- 降低运行维护费用
- 提高网络管理质量
- 保证业务质量
- 解决多厂商设备连通问题
- 综合已有和未来的技术
- 可持续发展

### TMN定义

ITU-T在20世纪80年代根据OSI系统管理框架提出的用于电信管理网络的一组**标准协议**规定的网络结构。提供一个有组织的体系结构，实现各个运行系统(OS)以及电信设备之间的互连，利用**标准接口**交换管理信息。

### TMN与电信网的关系（作业总结）

- **电信网**：负责承载用户业务（业务流）
- **TMN**：在电信网之上**逻辑独立**的管理网络，负责承载管理信息（管理流），对电信网进行集中监控和运行维护

### TMN体系结构三大方面

1. TMN**功能**体系结构（六大功能块+参考点）
2. TMN**信息**体系结构（参考点、信息模型、信息元素、交互模型）
3. TMN**物理**体系结构（六大物理元素+标准接口）

### NSA模型（通信网通用分层模型）

Network（网络层）→ Service（业务层）→ Application（应用层）

## 5.6 TMN功能体系结构（重点，2023论述2、2024论述5）

### TMN六大功能块（必背）

| 功能块 | 全称 | 核心功能 | 备注 |
|--------|------|---------|------|
| **OSF** | 运营系统功能 (Operations System Function) | **执行TMN管理功能(FCAPS)**，管理功能的"大脑" | 分为商务OSF、业务OSF、网络OSF、基层OSF四类 |
| **NEF** | 网元功能 (Network Element Function) | 使网元得到检测和控制，与TMN通信；是被管资源的抽象 | 位于被管电信网一侧 |
| **WSF** | 工作站功能 (Workstation Function) | 将管理信息(f接口)转换为用户可理解的形式(g接口)；人机界面 | f→g的翻译 |
| **MF** | 中介功能 (Mediation Function) | 协议变换、消息变换、地址映射、路由选择、集线、信息过滤/存储/选择 | 两个信息模型不同的功能块间的"中介" |
| **QAF** | Q适配器功能 (Q Adapter Function) | 完成**TMN和非TMN**之间的连接和信息转换 | 连接非TMN标准接口的设备 |
| **DCF** | 数据通信功能 (Data Communication Function) | 提供信息传输机制、路由选择、中继和互通 | 实现OSI 1-3层功能 |

### TMN六大功能成分

MAF（管理应用功能）、MIB（管理信息库）、ICF（信息转换功能）、PF（表示功能）、HMA（人机适配）、MCF（消息通信功能）

### TMN五类参考点（2024简答10直接考）

| 参考点 | 连接对象 | 功能 | 重要性 |
|--------|---------|------|--------|
| **q3** | OSF ↔ MF/NEF/QAF/OSF | 全面管理信息交换，最完整的标准化接口 | ★★★ 最重要 |
| **qx** | MF ↔ MF/QAF 等 | 局部管理信息交换，功能是q3的子集 | ★★ |
| **f** | WSF ↔ OSF/MF | 工作站物理连接，传输管理信息到WSF | ★★ |
| **x** | OSF ↔ OSF（不同TMN域） | 不同运营商/管理域间TMN互联 | ★★ |
| **g** | WSF ↔ 用户（TMN之外） | 人机交互界面 | ★ |
| **m** | QAF ↔ 非TMN被管实体 | 非TMN标准接口适配 | ★ |

                    │   OSF   │←——→ x参考点（其它TMN域的OSF）

                 q3  │   │  f
          ┌──────────┘   └─────────┐
     ┌────▼────┐              ┌────▼────┐
     │   MF    │              │   WSF   │

   qx  │  │ q3                    │ g
 ┌─────┘  └──────┐           ┌────▼────┐
 │               │           │  用户   │

 │          └────┬────┘
 │               │ m

```

## 5.8 TMN优点与缺点

**优点**：

- 信息模型标准化，统一多厂家设备规范管理
- 接口标准化（Q3/Qx/F/X），易于互联互通
- TMN独立于电信网，管理信息与业务流分离

**缺点**：

- 数据处理量过大、处理时延长
- 基于网元立场而非全程全网角度（管理粒度问题）
- 与CMIP协议密切相关，技术复杂
- 可靠性要求太高（管理网本身也需被管理）

## 5.9 SNMP网络管理

### SNMP特点

- 应用层协议，TCP/IP协议族的一部分
- 通过UDP来操作（端口161/162）
- 简单、轻量级、易于实现
- Internet网络管理的实际标准

### SNMP模型

- **Manager**（管理站/Client）：发出Get/Set请求，接收Agent响应
- **Agent**（代理）：响应Manager请求；也可主动向Manager发送**Trap**（非请求消息，如故障告警）
- 请求/响应模式 + 主动Trap告警

### SNMP vs CMIP

| 维度 | SNMP | CMIP |
|------|------|------|
| 来源 | IETF | ISO |
| 复杂度 | 简单 | 复杂 |
| 基础网络 | TCP/IP | OSI |
| 应用范围 | 广泛（Internet实际标准） | 有限（电信领域） |
| 传输层 | UDP | OSI协议栈 |
| 操作 | Get/Set/Trap | M-GET/M-SET/M-CREATE/M-DELETE等 |

### SNMP基本操作

1. **GET**：Manager向Agent请求特定被管对象的值
2. **GET-NEXT**：遍历MIB树中的下一个对象
3. **SET**：Manager向Agent设置被管对象的值
4. **TRAP**：Agent主动向Manager发送告警通知

## 2023年真题答案要点

### 简答题（每题5分）

1. **No.7信令链路功能**：属于MTP第2级。功能包括：分界定位（用标志码Flag标识信令单元边界）、差错检测（CRC校验）和校正（重传）、故障恢复（链路故障后自动恢复）、流量控制（防止发送过快淹没接收方）。

2. **传送网及其包含的内容**：传送网为交换节点/业务节点之间提供信息透明传输通道，独立于业务网。包含：传输介质（双绞线/同轴/光纤/无线）、交换设备（二层/三层/多层交换机）、技术体制（PDH→SDH→OTN演进）。

3. **广域网路由和互联网路由**：广域网路由解决一个子网内的路由选择，常用算法包括Dijkstra（链路状态）和距离矢量算法，代表协议OSPF/RIP。互联网路由解决不同自治系统/子网之间的路由，常用BGP协议（路径矢量路由）。

4. **通信网络可信化**：网络和用户行为结果可预期与可管理。包含三大属性：安全性（防攻击/加密）、可生存性（故障时仍提供基本服务）、可管理性（可监控/可配置）。具体包括服务提供者的可信、网络信息传输的可信、终端用户的可信。

5. **RSVP（资源预留协议）**：IntServ的信令协议，基本思想是资源预留。采用接收方控制的带内信令方式：发送方沿数据路径发送PATH报文（携带业务流规格TSpec），沿途路由器记录路径状态；接收方收到后沿反向路径回复RESV报文，在各路由器中预留所需资源。

6. **下游标签分配方式（MPLS）**：由LSP末端节点向上游方向分配标签。分为两种方式：(1)下游按需分发(DOD)——上游LSR明确请求后，下游才分配标签；(2)下游自主分发(DU)——下游LSR主动向所有上游分发<Label,FEC>绑定。优点是分配沿数据流方向，不会产生上游重复使用问题。

7. **智能网结构示意图**：画出三层结构：传输层（交换机、SSP、IP、中继）、信令层（No.7信令网、STP）、智能层（SCP、SMS、SCE），标注各层连接关系。

8. **软交换概念、特点及在NGN中的作用**：概念——将呼叫控制功能从媒体网关中分离出来，通过软件实现呼叫控制。特点：呼叫控制与承载分离、业务控制与呼叫控制分离、纯软件实现。作用：NGN的呼叫控制核心、协议适配枢纽、业务交换平台、异构网络互通桥梁。

## 2025年真题答案要点

### 简答题（每题5分）

1. **数据传输链路及其两类链路**：数据传输链路在物理传输基础上增加了检测、确认和重传功能，实现可靠传输。两类：(1)物理传输链路——仅能传送比特流但不可靠；(2)数据传输链路——具备差错控制和流量控制，可靠传输。

2. **支撑网的功能**：支撑网不直接传送用户业务，而是为业务网的正常运行提供控制、管理和同步功能的"幕后网络"。包含三大支撑：信令网（传递控制信息，No.7信令）、同步网（提供基准定时信号和时钟同步）、管理网（监测和控制网络运行，TMN/SNMP）。

3. **Dijkstra路由算法基本原理**：逐步扩展已确定最短路径的节点集合N'，直至包含所有节点。步骤：初始化N'={源节点}→循环：选不在N'中D(w)最小的w加入N'→对w的每个不在N'中的邻居v，更新D(v)=min(D(v), D(w)+c(w,v))。复杂度O(n²)。

4. **分组交换技术工作原理**：采用存储转发方式。发送端将数据分成若干分组（每个分组含目的地址和控制信息）；各分组独立在网络中传输，经过中间节点时先存储再查表转发；到达目的端后按序号重组恢复原始数据。特点：高效(统计复用)、灵活(独立选路)、有额外开销。

5. **接纳控制（IntServ功能部件）**：基于用户的服务协议(SLA/SLS)对用户访问网络进行监视和控制。每次新流请求到达时，判断在满足其QoS需求的前提下是否会降低已有流的QoS——若不会则接受并为该流预留资源，若会则拒绝。是IntServ实现QoS保证的关键部件。

6. **DiffServ业务类型**：EF(加速转发/优质业务——低延迟低抖动低丢包，保证带宽)、AF(确保转发/确保业务——4等级x3丢弃优先级共12个码点，预定最小保证值，超出按优先级丢包，具有动态QoS)、BE(尽力而为——无QoS保证)。

7. **智能网概念模型各平面功能**：SP(业务平面——面向用户定义智能网业务和业务特征)；GFP(全局功能平面——面向业务设计者，用SIB+GSL描述业务逻辑)；DFP(分布功能平面——面向网络设计者，定义9个功能实体FE和信息流IF)；PP(物理平面——面向网络实施者，将FE映射到实际物理设备，定义INAP接口)。

8. **基于软交换的NGN分层结构图**：画四层图——接入层(MG/SG/IAD)→传送层(IP骨干网)→控制层(软交换/MGC)→业务层(AS/SCP/AAA等)，标注各层协议。

9. **卫星地球站分系统**：负责发射用户信号到卫星和接收卫星转发的信号。组成：天线系统（抛物面天线/相控阵天线）、发射设备（上变频+功率放大）、接收设备（低噪声放大+下变频+解调）。中央站(Central Station/Hub)还负责整个卫星网络的业务调度、管理和监测控制。

### 论述题（每题10分）

1. **局间接续基本信令流程**：以两个端局间电话呼叫为例，画出ISUP信令时序图：主叫局→IAM(初始地址消息)→被叫局；被叫局→ACM(地址全消息)→主叫局(被叫振铃)；被叫局→ANM(应答消息)→主叫局(被叫摘机)；...通话...；任一方→REL(释放消息)→对方；对方→RLC(释放完成)→释放方。标注每个消息的含义和作用。

2. **距离矢量与链路状态路由算法**：
   - 距离矢量(DV)：各节点维护到所有目的节点的距离向量；仅与邻居交换DV信息；基于Bellman-Ford方程迭代更新Dx(y)=min_v{c(x,v)+Dv(y)}；存在计数到无穷问题，需毒性逆转解决；代表协议RIP/BGP。
   - 链路状态(LS)：各节点通过全网洪泛获知完整拓扑；独立用Dijkstra算法计算最短路径树；收敛快、可靠性高；代表协议OSPF/IS-IS。
   - 对比：从信息交换范围、收敛速度、开销、扩展性、可靠性等维度对比。

3. **探月工程卫星通信原理**（四段式）：
   - 背景：探月工程（嫦娥四号/六号月背探测），体现航天发射、深空测控、中继通信等综合科技实力。
   - 技术：采用中继卫星（鹊桥一号L2点Halo轨道/鹊桥二号环月大椭圆冻结轨道），S/X频段通信。
   - 原理：巡视器/着陆器在月背→S频段信号→鹊桥中继卫星透明转发→X频段→地球深空测控站→地面网络。由于月球本体遮挡，若无中继卫星则月背探测器无法直接对地通信。
   - 意义：全球首次实现月背直接通信，体现深空探测通信能力，为后续探月任务（嫦娥七号/八号）和载人登月提供通信基础设施。

4. **对5G/6G/WSN的理解**（选其一展开，四段式）：
   - 选6G：6G愿景(1Tbps, 0.1ms, 10x频谱效率)+关键技术(太赫兹/AI原生/通感算一体/数字孪生/空天地海一体化)+中国成就（5G领先+6G标准主导+千帆星座+华为专利）
   - 选WSN：基本架构(传感器→汇聚→网关→云)+关键协议(Zigbee/LoRaWAN/NB-IoT)+能耗瓶颈(LEACH分簇路由)+应用(环境监测/精准农业/工业物联网)
   - 内大智慧校园应用：远程教学/智慧图书馆/绿色校园/安防/草原牧区
   - 展望：6G/WSN将推动社会数字化转型

5. **MPLS组网基本原理**：
   - 核心思想：用标签替代IP查表，将路由控制与分组转发分离。
   - 组网过程：(1)LDP+IGP建立标签转发表和LSP；(2)入口LER识别FEC→压入标签(PUSH)；(3)核心LSR仅查标签表→交换标签(SWAP)→高速转发；(4)出口LER弹出标签(POP)→恢复IP转发。
   - 关键组件：LER(边缘，PUSH/POP)、LSR(核心，SWAP)、LSP(标签交换路径)、LDP(标签分发协议，分发<Label,FEC>映射)、FEC(等价转发类)。
   - MPLS头结构：20bit Label+3bit EXP+1bit S+8bit TTL=32bit，位于L2和L3之间（"2.5层"）。
   - 优点：转发快、支持TE流量工程、支持VPN、QoS保障。
